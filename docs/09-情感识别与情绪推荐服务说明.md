# 情感识别与情绪推荐服务说明

## 1. 概述

`services/analysis_service.py` 中的情感识别模块，负责将用户输入的自然语言文本（如 "今天心情很糟糕"、"想看点轻松搞笑的"）自动识别为**预定义的情绪场景**，并据此推荐最契合当前情绪状态的电影。

该模块采用 **Zero-Shot Classification（零样本文本分类）** 技术，无需为每种情绪单独标注训练数据，即可将任意文本映射到 7 种人文关怀型情绪标签中，兼具技术效率与产品温度。

---

## 2. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 模型框架 | **Hugging Face Transformers** | Python 生态最成熟的 NLP 库，pipeline API 极简封装 |
| 分类范式 | **Zero-Shot Classification** | 无需情绪标注语料，利用预训练模型的语义理解能力直接推断文本意图 |
| 本地模型 | **Chinese-XLM-XNLI**（或兼容的本地化多语言模型） | 支持中文文本理解，可离线部署，无需调用云端 API |
| 缓存策略 | **functools.lru_cache** | 模型体积大（数百 MB 至数 GB），通过单例缓存避免重复加载 |

---

## 3. 情绪场景配置 (`MOOD_SCENARIOS`)

系统内置 **7 种情绪场景**，每种场景包含：
- `genres`：主要推荐的电影类型
- `healing`（可选）：治愈/平衡类电影类型，用于负面情绪的双轨推荐策略
- `msg`：带 Emoji 的人文关怀文案，在推荐结果中展示给用户

| 情绪标签 | 主要类型 | 治愈类型 | 推荐策略 | 关怀文案示例 |
|----------|----------|----------|----------|--------------|
| 😄 开心 | `Comedy`, `Animation`, `Family`, `Musical` | — | 快乐延续，加倍推荐轻松喜剧 | "保持这份好心情！这些电影会让笑容延续" |
| 😭 难过 | `Drama`, `Romance` | `Family`, `Animation`, `Comedy` | 50% 共情 + 50% 治愈 | "抱抱你。哭出来没关系，让这些温暖的故事陪陪你" |
| 😤 愤怒 | `Action`, `Crime`, `War` | `Documentary`, `Music`, `Biography` | 发泄 + 冷静双轨 | "深呼吸。去电影里宣泄压力，或在真实故事找回平静" |
| 😨 害怕 | `Horror`, `Thriller`, `Mystery` | `Adventure`, `Comedy`, `Fantasy` | 以毒攻毒 + 壮胆 | "别怕，光影与你同在。选一部冒险喜剧找回勇气" |
| 😎 刺激 | `Action`, `Adventure`, `Sci-Fi` | — | 肾上腺素飙升 | "准备好起飞了吗？系好安全带" |
| 🧘 平静 | `Documentary`, `Biography`, `History`, `Music` | — | 慢节奏、心灵漫步 | "享受这段独处的时光，让心灵在光影中漫步" |
| 🤔 烧脑 | `Mystery`, `Sci-Fi`, `Crime` | — | 迷宫式谜题挑战 | "大脑开始运转！准备好挑战这些谜题了吗？" |

> **设计亮点**：负面情绪（难过、愤怒、害怕）均采用 **双轨策略**——既提供情绪出口（如看惊悚片宣泄），也提供治愈选项（如看动画找回温暖），避免单一推荐带来的不适感。

## 4. 模型加载机制 (`get_nlp_classifier`)

```python
@functools.lru_cache(maxsize=1)
def get_nlp_classifier()
```

**关键设计**：
- **单例缓存**：使用 `@functools.lru_cache(maxsize=1)` 确保全局仅加载一次模型，避免内存重复占用
- **本地部署**：模型存放于 `项目根目录/ml_models/chinese_xlm_xnli/`，包含 `pytorch_model.bin` 等标准 HuggingFace 文件
- **容错处理**：若模型文件缺失或加载失败，打印错误日志并返回 `None`，上层调用自动降级

模型路径配置：
```python
LOCAL_MODEL_PATH = os.path.join(BASE_DIR, "ml_models", "chinese_xlm_xnli")
```

## 5. 情感识别核心逻辑 (`analyze_text_mood`)

```python
def analyze_text_mood(text: str) -> str | None
```

**输入输出**：
- **输入**：用户原始文本，如 `"今天心情很糟糕，想找点治愈的"`
- **输出**：带 Emoji 的情绪 Key，如 `😭 难过`，或 `None`（文本过短/置信度不足）

**执行流程**：

1. **空值过滤**：文本为空或少于 2 个字符时直接返回 `None`
2. **提取候选标签**：从 `MOOD_SCENARIOS` 的 Keys 中剥离 Emoji，得到纯文本标签列表：
   ```python
   ['开心', '难过', '愤怒', '害怕', '刺激', '平静', '烧脑']
   ```
3. **模型推理**：调用 `pipeline("zero-shot-classification")`：
   ```python
   result = classifier(text, candidate_labels, multi_label=False)
   ```
   - `multi_label=False`：强制单标签输出，确保情绪归属唯一
4. **置信度阈值**：若最高置信度 `< 0.2`，视为识别失败，返回 `None`
5. **映射还原**：将纯文本标签还原为带 Emoji 的完整 Key

**推理示例**：
```
分析结果: '今天心情不好，想大哭一场' -> 难过 (置信度: 0.89)
分析结果: '无聊想看点高智商的' -> 烧脑 (置信度: 0.76)
分析结果: 'abc' -> None (置信度: 0.12 < 0.2)
```

## 6. 情绪推荐逻辑 (`get_movies_by_mood`)

```python
async def get_movies_by_mood(mood_key: str, limit=12, category='all')
```

**执行流程**：

### 6.1 解析场景配置
- 根据 `mood_key` 从 `MOOD_SCENARIOS` 获取配置
- 合并 `genres`（主要类型）与 `healing`（治愈类型）为 `target_genres`
- 提取暖心文案 `warm_msg`

### 6.2 数据库查询

```python
conditions = [TitleBasics.genres.ilike(f"%{g}%") for g in target_genres]
query = (
    select(TitleBasics.primaryTitle, TitleRatings.averageRating, ...)
    .join(TitleRatings, ...)
    .where(or_(*conditions))
    .where(TitleRatings.numVotes > 5000)  # 只取高分热门
)
```

- **类型匹配**：使用 SQL `ILIKE` 模糊匹配，支持多类型组合字段（如 `Action,Adventure`）
- **质量门槛**：`numVotes > 5000`，过滤掉冷门低质影片
- **分类过滤**：支持 `movie`/`tv`/`anime`/`variety`/`doc` 二次过滤（同推荐服务统一规则）

### 6.3 随机打乱策略

```python
query = query.order_by(desc(TitleRatings.averageRating)).limit(50)
random.shuffle(all_movies)
```

- 先按评分降序取 **Top 50** 高质量候选池
- 再在**内存中随机打乱**，每次推荐结果不同，制造"偶遇"感
- 最终截取前 `limit` 条（默认 12 条）返回

### 6.4 返回值

```python
return all_movies[:limit], warm_msg
```

- **`all_movies`**：推荐电影列表，每条包含 `primaryTitle`、`averageRating`、`genres`、`startYear`、`tconst`
- **`warm_msg`**：该情绪场景的人文关怀文案，前端可直接展示

## 7. 调用方式

### 情感识别（文本 → 情绪标签）

```python
from services.analysis_service import analyze_text_mood

mood_key = analyze_text_mood("最近工作压力大，想发泄一下")
print(mood_key)  # 😤 愤怒
```

### 情绪推荐（情绪标签 → 电影列表）

```python
from services.analysis_service import get_movies_by_mood

movies, msg = await get_movies_by_mood("😤 愤怒", limit=8, category='movie')
print(msg)  # 🍃 深呼吸。去电影里宣泄压力...
for m in movies:
    print(f"  - {m['primaryTitle']} ({m['averageRating']})")
```

## 8. 与其他模块的关系

```
用户输入文本
    │
    ▼
[analyze_text_mood] Zero-Shot 情感识别
    │
    ├── 置信度 < 0.2 ──→ 返回 None（前台降级到热门推荐）
    │
    └── 置信度 >= 0.2 ──→ 返回 mood_key（如 😭 难过）
                │
                ▼
        [get_movies_by_mood] 情绪推荐
                │
                ├── 查询 target_genres 匹配的电影
                ├── 过滤 numVotes > 5000
                ├── 应用 category 二次过滤
                ├── Top 50 评分排序
                ├── 内存随机打乱
                └── 返回电影列表 + warm_msg
```

## 9. 注意事项

1. **模型文件依赖**：首次使用前必须将 `chinese_xlm_xnli` 模型文件放入 `ml_models/chinese_xlm_xnli/` 目录，至少包含 `pytorch_model.bin`、`config.json`、`tokenizer.json`
2. **硬件要求**：Zero-Shot 模型通常基于 Transformer（如 XLM-RoBERTa），推理时建议至少 2GB 空闲内存；GPU 可加速但不强制
3. **冷启动降级**：模型加载失败或识别置信度不足时，系统返回 `None`，前台应优雅降级到**热门推荐**或**手动选择情绪**
4. **中文适配**：当前配置面向中文用户输入，若需扩展其他语言，应替换为多语言模型（如 `xlm-roberta-large-xnli`）并同步更新 `candidate_labels`
5. **情绪标签硬编码**：`MOOD_SCENARIOS` 为内置字典，如需新增情绪（如 "浪漫"、"孤独"），需同步修改 `genres`、`healing`、`msg` 三要素
6. **缓存刷新**：模型加载后驻留内存直到进程重启；若更新模型文件，需重启服务使 `lru_cache` 失效重新加载
