# 离线推荐引擎说明

## 1. 概述

`spark_runner.py` 是项目的**离线推荐任务脚本**，负责基于用户历史评分数据，利用 **PySpark + ALS（交替最小二乘法）** 训练协同过滤模型，为每个用户生成 Top-10 个性化电影推荐，并将结果同步回 PostgreSQL 数据库，供前端实时查询展示。

由于该任务计算量较大（涉及矩阵分解与全量用户推理），采用 **离线批处理 + 结果入库** 的模式运行，而非实时在线预测。

---

## 2. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 计算引擎 | **PySpark** | 内置 ALS 实现，支持分布式矩阵分解，可轻松处理万级用户/电影规模 |
| 模型算法 | **ALS (Alternating Least Squares)** | Spark MLlib 原生支持，适合显式评分（explicit feedback）场景，收敛稳定 |
| 数据交换 | **Pandas + SQLAlchemy** | 负责与 PostgreSQL 的轻量级读写桥接，无缝衔接 Spark DataFrame |
| 数据存储 | **PostgreSQL** | 复用项目统一数据库，推荐结果写入独立表，避免与业务数据耦合 |

## 4. 核心流程

整个离线 pipeline 分为 9 个阶段，下面按执行顺序逐一说明。

### 4.1 初始化 Spark Session

```python
spark = SparkSession.builder \
    .appName("Movie_Recommendation_ALS_Offline") \
    .config("spark.driver.memory", "4g") \
    .master("local[*]") \
    .getOrCreate()
```

- **应用名称**：`Movie_Recommendation_ALS_Offline`，便于在 Spark UI 中识别。
- **驱动内存**：分配 `4g`，保证中等规模数据集的矩阵分解不溢出。
- **运行模式**：`local[*]`，利用本地所有 CPU 核心进行并行计算。如需扩展，可无缝切换为 `yarn` 或 `standalone` 集群模式。
- **日志级别**：设置为 `ERROR`，减少控制台无关输出。

### 4.2 数据摄取

```python
query = "SELECT user_id, tconst, rating FROM user_personal_ratings"
pdf_ratings = pd.read_sql(query, engine)
```

从 `user_personal_ratings` 表读取用户评分三元组：
- `user_id`：用户唯一标识（数值型）
- `tconst`：IMDb 影视唯一标识（字符串型，如 `tt0110912`）
- `rating`：用户给出的评分（通常为 0-10 或 1-5 分制）

若读取结果为空，脚本立即终止并输出警告，防止后续空数据训练报错。

### 4.3 特征工程（StringIndexer）

```python
string_indexer = StringIndexer(inputCol="tconst", outputCol="movie_idx")
indexer_model = string_indexer.fit(ratings_df)
indexed_ratings = indexer_model.transform(ratings_df)
```

ALS 模型要求 **itemCol 为数值型**，因此使用 Spark ML 的 `StringIndexer` 将 `tconst` 映射为连续的整数索引 `movie_idx`。训练后的 `indexer_model` 会在后续步骤中用于逆向还原。

### 4.4 划分训练集与测试集

```python
(training, test) = indexed_ratings.randomSplit([0.8, 0.2], seed=42)
```

- **比例**：80% 训练 / 20% 测试
- **随机种子**：`seed=42`，保证每次执行划分结果一致，便于复现实验与横向对比调参效果。

### 4.5 模型训练

```python
als = ALS(
    rank=4,
    maxIter=15,
    regParam=0.5,
    userCol="user_id",
    itemCol="movie_idx",
    ratingCol="rating",
    coldStartStrategy="drop",
    nonnegative=True
)
model = als.fit(training)
```

关键超参数说明：

| 参数 | 取值 | 含义 |
|------|------|------|
| `rank` | `4` | 隐因子（latent factor）维度，决定用户/电影特征向量的长度 |
| `maxIter` | `15` | 交替最小二乘的最大迭代次数 |
| `regParam` | `0.5` | L2 正则化系数，防止过拟合 |
| `coldStartStrategy` | `"drop"` | 遇到训练集中未出现的用户或电影时，直接丢弃该行，避免预测为 NaN |
| `nonnegative` | `True` | 限制隐因子非负，使评分预测更符合语义（评分通常不为负） |

### 4.6 模型评估（RMSE）

```python
evaluator = RegressionEvaluator(
    metricName="rmse",
    labelCol="rating",
    predictionCol="prediction"
)
rmse = evaluator.evaluate(predictions)
```

使用 **均方根误差（Root Mean Squared Error）** 作为核心评估指标：

$$
RMSE = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2}
$$

- **$y_i$**：用户真实评分
- **$\hat{y}_i$**：模型预测评分
- **$n$**：测试集样本数

脚本内置经验阈值判断：
- **RMSE > 1.2**：提示指标偏高，建议调整 `rank` / `regParam` 或检查数据噪声
- **RMSE ≤ 1.2**：提示模型性能表现良好

### 4.7 生成全局推荐

```python
user_recs = model.recommendForAllUsers(10)
```

利用训练好的 ALS 模型，为**所有用户**并行计算 Top-10 最可能感兴趣的电影。该操作在 Spark 内部通过矩阵乘法高效完成，无需逐用户循环。

### 4.8 逆向还原索引与数据平铺

```python
recs_exploded = user_recs.select(
    col("user_id"),
    explode("recommendations").alias("rec")
).select(
    col("user_id"),
    col("rec.movie_idx").alias("movie_idx"),
    col("rec.rating").alias("score")
)
```

- **`explode`**：将 `recommendations` 数组列展开为多行，每行对应一条推荐记录
- **`IndexToString`**：利用第 4.3 步保存的 `indexer_model.labels`，将 `movie_idx` 还原为原始的 `tconst`

最终字段：`user_id`、`tconst`、`score`（预测评分/置信度）。

### 4.9 结果入库

```python
with engine.connect() as conn:
    conn.execute(text("TRUNCATE TABLE spark_recommendations"))
    conn.commit()

result_pdf.to_sql('spark_recommendations', engine, if_exists='append', index=False)
```

- **先清空**：使用 `TRUNCATE` 确保每次推荐结果全量替换，避免旧推荐与新推荐混存
- **再写入**：通过 `to_sql` 将 Pandas DataFrame 批量插入 `spark_recommendations` 表
- **事务安全**：`TRUNCATE` 与 `append` 两步之间若发生异常，可在外层补充更严谨的事务控制（当前实现为简化版）

## 7. 调优建议

若 RMSE 偏高或推荐效果不理想，可尝试以下调优方向：

1. **增大 `rank`**：提升模型表达能力，但会增加训练时间与内存占用
2. **调整 `regParam`**：增大正则化可抑制过拟合，减小可提升拟合度
3. **增加 `maxIter`**：让模型充分收敛，但边际收益通常递减
4. **数据层面**：清洗异常评分、补充更多用户行为数据（如浏览、收藏等隐式反馈）
5. **冷启动处理**：新用户/新电影可考虑结合基于内容的推荐作为兜底策略

## 8. 注意事项

- **内存要求**：`spark.driver.memory=4g` 为中等规模设定；若用户数或电影数激增（百万级），请提升该配置或改为集群模式
- **全量替换策略**：当前 `TRUNCATE + append` 会在任务执行期间短暂清空推荐表；若对前端可用性要求极高，建议改为**先写入临时表，再重命名交换**
- **调度执行**：建议将该脚本接入定时任务（如 Linux `cron`、Windows 计划任务、或 CI/CD 流水线），按天/小时级频率刷新推荐结果
