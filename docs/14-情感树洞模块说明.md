# 情感树洞模块说明

## 1. 概述

情感树洞是影视推荐系统的**情感化交互功能**，通过自然语言处理技术识别用户当前情绪，结合暖心文案推荐匹配的影视作品。该模块将用户的主观感受与电影推荐深度结合，体现了系统的人文关怀特性。

---

## 2. 核心功能

### 2.1 功能特性

- **自然语言情绪识别**：用户输入任意文字，系统自动分析情绪
- **7种情绪场景**：覆盖开心、难过、愤怒、害怕、刺激、平静、烧脑
- **暖心文案**：根据情绪提供人文关怀的推荐语
- **智能推荐**：基于情绪类型匹配电影类型，支持治愈策略
- **手动选择**：提供情绪标签快捷选择，无需文字输入
- **分类过滤**：支持按电影/电视剧/动漫/综艺/纪录片过滤

### 2.2 用户交互流程

```
用户输入文字（如"今天加班好累"）
    │
    ├─→ NLP 模型分析情绪
    │       └─→ 识别为"难过"
    │
    ├─→ 查询匹配电影
    │       └─→ Drama + Romance + 治愈类
    │
    └─→ 返回推荐结果 + 暖心文案
            │
            └─→ 展示弹窗："🌻 抱抱你。哭出来没关系..."
```

---

## 3. 技术架构

### 3.1 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| **NLP 模型** | `chinese_xlm_xnli` (Hugging Face) | 支持中文 Zero-Shot 分类，无需训练 |
| **推理框架** | `transformers.pipeline` | 简化模型加载与推理流程 |
| **模型存储** | 本地目录 `ml_models/chinese_xlm_xnli` | 避免网络延迟，提升响应速度 |
| **缓存机制** | `@functools.lru_cache(maxsize=1)` | 模型单例加载，避免重复初始化 |
| **数据库查询** | SQLAlchemy AsyncSession | 异步查询，不阻塞主线程 |

### 3.2 模块结构

```
services/analysis_service.py
    ├── MOOD_SCENARIOS          # 情绪场景配置
    ├── get_nlp_classifier()    # NLP 模型加载（带缓存）
```

## 5. NLP 情感识别

### 5.1 模型加载

```python
@functools.lru_cache(maxsize=1)
def get_nlp_classifier():
    """单例模式加载 NLP 模型"""
    print(f"📂 [NLP] 正在加载本地模型: {LOCAL_MODEL_PATH}")
    if not os.path.exists(os.path.join(LOCAL_MODEL_PATH, 'pytorch_model.bin')):
        print("❌ 未找到模型文件！请确认路径正确。")
        return None
    try:
        classifier = pipeline("zero-shot-classification", model=LOCAL_MODEL_PATH, tokenizer=LOCAL_MODEL_PATH)
        print("✅ [NLP] 模型加载完成！")
        return classifier
    except Exception as e:
        print(f"❌ [NLP] 模型加载失败: {e}")
        return None
```

**关键点**：
- 使用 `@functools.lru_cache(maxsize=1)` 确保模型只加载一次
- 模型路径：`项目根目录/ml_models/chinese_xlm_xnli`
- 失败时返回 `None`，上层调用自动降级

### 5.2 情绪识别逻辑

```python
def analyze_text_mood(text: str):
    """意图识别：输入文字 -> 返回带 Emoji 的情绪 Key"""
    if not text or len(text.strip()) < 2:
        return None

    try:
        classifier = get_nlp_classifier()
        # 从配置中提取候选标签（去掉 Emoji）
        labels_map = {k.split(' ')[1]: k for k in MOOD_SCENARIOS.keys()}
        candidate_labels = list(labels_map.keys())

        # Zero-Shot 分类
        result = classifier(text, candidate_labels, multi_label=False)
        top_label = result['labels'][0]
        top_score = result['scores'][0]

        print(f"分析结果: '{text}' -> {top_label} (置信度: {top_score:.2f})")
        
        # 置信度阈值：低于 0.2 返回 None
        if top_score < 0.2:
            return None
        return labels_map.get(top_label)
    except Exception as e:
        print(f"❌ 模型推理失败: {e}")
        return None
```

**处理流程**：
1. 输入验证：文本长度需 ≥ 2
2. 标签提取：从 `MOOD_SCENARIOS` 提取纯文本标签（如"开心"、"难过"）
3. Zero-Shot 分类：模型计算文本与各标签的匹配度
4. 置信度过滤：低于 0.2 认为识别失败
5. 结果映射：返回带 Emoji 的完整 Key（如"😭 难过"）

### 5.3 Zero-Shot 分类原理

**Zero-Shot Learning** 允许模型在没有见过特定训练数据的情况下，对未知类别进行分类。

本模块使用的 `chinese_xlm_xnli` 模型基于自然语言推理（NLI）任务训练：
- 将输入文本与候选标签构造为"前提-假设"对
- 模型判断"假设"是否可以从"前提"中推断
- 推断概率最高的标签即为识别结果

**示例**：
```
输入："今天加班好累"
假设1："用户感到开心" → 矛盾 (低分)
假设2："用户感到难过" → 蕴含 (高分)
假设3："用户感到愤怒" → 中立 (中分)
→ 结果：难过
```

## 6. 情绪推荐逻辑

### 6.1 推荐函数

```python
async def get_movies_by_mood(mood_key: str, limit=12, category='all'):
    """根据心情推荐电影 (混合推荐 + 返回文案)"""
    scenario = MOOD_SCENARIOS.get(mood_key)
    if not scenario:
        return [], ""

    # 1. 提取主要类型 和 治愈类型
    primary_genres = scenario.get('genres', [])
    healing_genres = scenario.get('healing', [])

    # 2. 混合策略：如果有治愈类型，扩大查询范围
    target_genres = primary_genres + healing_genres

    # 3. 获取暖心文案
    warm_msg = scenario.get('msg', '愿电影陪你度过美好时光。')

    async with AsyncSessionLocal() as db:
        # 构建基础查询
        conditions = [TitleBasics.genres.ilike(f"%{g}%") for g in target_genres]

        query = (
            select(TitleBasics.primaryTitle, TitleRatings.averageRating, 
                   TitleBasics.genres, TitleBasics.startYear, TitleBasics.tconst)
            .join(TitleRatings, TitleBasics.tconst == TitleRatings.tconst)
            .where(or_(*conditions))
            .where(TitleRatings.numVotes > 5000)  # 只要高分
        )

        # 应用分类过滤
        if category != 'all':
            query = _apply_category_filter(query, category)

        # 随机取样策略：取 50 部高分电影，内存打乱
        query = query.order_by(desc(TitleRatings.averageRating)).limit(50)

        result = await db.execute(query)
        all_movies = []
        for row in result.all():
            all_movies.append({
                'primaryTitle': row.primaryTitle,
                'averageRating': row.averageRating,
                'genres': row.genres,
                'startYear': row.startYear,
                'tconst': row.tconst
            })

        # 内存中随机打乱，实现"偶遇"感
        random.shuffle(all_movies)

        # 返回截取后的列表 和 暖心文案
        return all_movies[:limit], warm_msg
```

### 6.2 推荐策略

#### 类型匹配

- **主要类型**：直接匹配情绪对应的类型（如"难过"匹配 Drama、Romance）
- **治愈类型**：负面情绪额外匹配治愈类型（如"难过"额外匹配 Family、Animation、Comedy）
- **OR 条件**：满足任一类型即可，扩大召回范围

#### 质量过滤

- **投票数过滤**：`numVotes > 5000`，确保推荐有一定热度的电影
- **评分排序**：按 `averageRating` 降序取 Top 50
- **随机打乱**：内存中 `random.shuffle()`，每次推荐不同，增加"偶遇"感

#### 分类过滤

支持按影视类型二次过滤：

| 分类 | 过滤条件 |
|------|----------|
| `'all'` | 不过滤 |
| `'movie'` | `titleType` 为 `movie` 或 `tvMovie`，排除纪录片 |
| `'tv'` | `titleType` 为 `tvSeries` 或 `tvMiniSeries` |
| `'anime'` | `genres` 包含 `Animation` |
| `'variety'` | `genres` 包含 `Reality-TV`、`Talk-Show` 或 `Game-Show` |
| `'doc'` | `genres` 包含 `Documentary` |

### 6.3 返回结果

返回元组：`(电影列表, 暖心文案)`

## 9. 容错与降级

### 9.1 模型加载失败

- **检测**：检查 `pytorch_model.bin` 是否存在
- **处理**：返回 `None`，上层调用自动降级
- **日志**：打印错误信息，便于排查

### 9.2 情绪识别失败

- **置信度不足**：`top_score < 0.2` 时返回 `None`
- **模型推理异常**：捕获异常，返回 `None`
- **降级策略**：前端提示用户"换个说法"或使用手动标签选择

### 9.3 推荐结果为空

- **数据库无匹配数据**：返回空列表
- **UI 处理**：显示安慰文案"暂未找到匹配的影片，不过没关系，休息一下也是很好的选择"
- **兜底建议**：引导用户使用其他推荐方式（如热门榜单）

## 10. 性能优化

### 10.1 模型缓存

使用 `@functools.lru_cache(maxsize=1)` 确保模型只加载一次：

```python
@functools.lru_cache(maxsize=1)
def get_nlp_classifier():
    # 模型加载逻辑
```

**效果**：
- 首次调用：加载模型（约 2-3 秒）
- 后续调用：直接返回缓存对象（< 1ms）

### 10.2 数据库查询优化

- **索引优化**：`TitleBasics.genres`、`TitleRatings.numVotes` 建立索引
- **限制数量**：先取 Top 50，内存中打乱后截取，避免大量数据传输
- **异步查询**：使用 `AsyncSession`，不阻塞主线程

### 10.3 前端响应优化

- **加载提示**：点击按钮后立即显示通知"正在感知您的情绪..."
- **本地缓存**：情绪标签配置在前端静态渲染，无需后端查询
- **懒加载**：弹窗内容按需加载，减少初始渲染压力
