# 爬虫模块说明

## 1. 概述

`crawlers/` 目录包含两个独立的数据抓取脚本，负责为项目补充 IMDb 官方数据集之外的外部数据：

- **`top250_crawler.py`** —— 抓取**豆瓣电影 Top 250** 榜单数据，用于中外评分对比可视化
- **`box_office_crawler.py`** —— 抓取**Box Office Mojo** 全球票房数据，用于票房与口碑分析

两个爬虫均基于 Python 标准库 + `BeautifulSoup` 进行 HTML 解析，直接写入 PostgreSQL 数据库，与项目 ORM 模型无缝集成。

---

## 2. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| HTTP 请求 | `requests` / `httpx` | `top250` 使用同步请求（页面少），`box_office` 使用异步请求（并发量大） |
| HTML 解析 | **BeautifulSoup4** | 结构简单，XPath 或正则无需引入额外复杂度 |
| 并发模型 | `asyncio` + `Queue` | 票房爬虫需处理数百次请求，8 并发异步 Worker 提升效率 |
| 数据写入 | **SQLAlchemy ORM + PostgreSQL Upsert** | 复用项目数据库连接，支持"有则更新、无则插入" |
| 反爬策略 | 随机延迟 + Cookie + User-Agent | 降低请求频率，模拟真实浏览器行为 |

## 3. 豆瓣 Top 250 爬虫 (`top250_crawler.py`)

### 3.1 抓取目标

- **来源**：https://movie.douban.com/top250
- **数据量**：250 部电影（10 页，每页 25 条）
- **更新频率**：建议每周运行一次，跟踪排名浮动

### 3.2 抓取字段

| 字段 | 来源 | 说明 |
|------|------|------|
| `rank` | 列表序号 `<em>` | 豆瓣 Top 250 排名（1-250） |
| `title` | 主标题 `<span class="title">` | 中文电影名称 |
| `douban_id` | 详情页 URL | 豆瓣内部 ID，如 `1292052` |
| `douban_score` | 评分 `<span class="rating_num">` | 豆瓣用户评分（0-10） |

> **注意**：`imdb_id` 字段在列表页不可见，**暂时留空**，需后续通过手动补全或二次爬虫回填，用于与 IMDb 数据精确关联。

### 3.3 反爬策略

```python
HEADERS = {
    'User-Agent': 'Mozilla/5.0 ...',
    'Referer': 'https://movie.douban.com/top250',
    'Cookie': MY_COOKIE  # 已登录用户的 Cookie
}
```

- **Cookie 认证**：豆瓣对未登录用户限制更严格，需填入已登录账号的 Cookie
- **随机延迟**：每页请求前 `sleep(1~3秒)`，避免触发频率限制
- **分页参数**：`?start=0`、`?start=25` ... `?start=225`

### 3.4 数据入库逻辑

采用**查询-更新/新增**模式：

1. 以 `douban_id` 为唯一键查询是否已存在
2. **若存在**：仅更新 `rank`、`title`、`douban_score`，**保留**已有的 `imdb_id`（防止手动补全数据被覆盖）
3. **若不存在**：插入新记录，`imdb_id` 留空

### 3.5 运行方式

```bash
python crawlers/top250_crawler.py
```

首次运行前务必检查 `MY_COOKIE` 变量，确保为有效登录态 Cookie（不含中文分号或空格）。

## 4. 票房数据爬虫 (`box_office_crawler.py`)

### 4.1 抓取目标

- **来源**：Box Office Mojo（https://www.boxofficemojo.com/title/{tconst}/）
- **数据量**：默认爬取 IMDb 投票数 Top 600 部电影的票房
- **更新频率**：建议每日或每周运行

### 4.2 抓取字段

| 字段 | 来源 | 说明 |
|------|------|------|
| `tconst` | 传入参数 | IMDb 唯一标识，如 `tt0111161` |
| `box_office` | `div.mojo-performance-summary-table` | 全球累计票房（美元） |
| `rank` | 程序计算 | 按 IMDb `numVotes` 降序的排名 |

**票房提取逻辑**：
1. 请求 `boxofficemojo.com/title/{tconst}` 页面
2. 解析 `mojo-performance-summary-table` 容器
3. 提取所有 `<span class="money">` 标签
4. 清洗货币格式（去掉 `$` 和 `,`），取**最大值**作为全球票房

### 4.3 异步并发架构

采用 **Producer-Consumer（生产者-消费者）模式**：

```python
queue = asyncio.Queue()
# Producer：将 600 部电影放入队列
for idx, (tconst, title) in enumerate(movies):
    queue.put_nowait((tconst, title, idx + 1))

# Consumer：启动 8 个 Worker 并发处理
async with httpx.AsyncClient(...) as client:
    tasks = [asyncio.create_task(worker(queue, client, db)) for _ in range(CONCURRENCY)]
    await queue.join()
```

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `TARGET_COUNT` | 600 | 抓取目标数量（IMDb 投票数 Top N） |
| `CONCURRENCY` | 8 | 并发 Worker 数量 |
| 随机延迟 | 0.5~1.5 秒 | 每次请求前随机等待，降低被封风险 |
| 超时 | 15 秒 | `httpx.AsyncClient(timeout=15.0)` |

### 4.4 数据入库：Upsert 策略

使用 PostgreSQL 的 **`ON CONFLICT DO UPDATE`** 实现原子级"存在即更新、不存在即插入"：

```python
from sqlalchemy.dialects.postgresql import insert

stmt = insert(MovieBoxOffice).values(
    tconst=tconst,
    box_office=money,
    rank=rank
).on_conflict_do_update(
    index_elements=['tconst'],
    set_=dict(box_office=money, rank=rank)
)
```

- **冲突键**：`tconst`（唯一约束）
- **更新字段**：`box_office`、`rank`
- **无需先查询再判断**，单条 SQL 完成全部逻辑，避免并发竞态

### 4.5 运行方式

```bash
python crawlers/box_office_crawler.py
```

## 5. 公共依赖与注意事项

### 5.1 环境准备

两个爬虫均需安装以下依赖：

```bash
pip install beautifulsoup4 requests httpx sqlalchemy psycopg2-binary lxml
```

> `lxml` 为 BeautifulSoup 的解析器后端，解析速度优于内置 `html.parser`。

### 5.2 数据库连接

爬虫脚本通过 `sys.path` 向上追溯项目根目录，导入项目自身的 `database.AsyncSessionLocal` 和 `models`：

```python
sys.path.append(os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
from database import AsyncSessionLocal
from models import DoubanTop250, MovieBoxOffice, TitleBasics, TitleRatings
```

因此运行时**必须**确保：
- 项目根目录的数据库配置（`database.py`）可正常连接 PostgreSQL
- 目标表已通过 `init_db.py` 或 Alembic 迁移创建

### 5.3 Windows 兼容性

两个脚本均包含 Windows 事件循环兼容补丁：

```python
if sys.platform == 'win32':
    asyncio.set_event_loop_policy(asyncio.WindowsSelectorEventLoopPolicy())
```

防止在 Windows 下使用 `asyncio` + `httpx`/`requests` 时出现 `NotImplementedError`。

### 5.4 反爬与风控建议

1. **Cookie 有效性**：豆瓣 Cookie 会过期，建议定期更新；若爬取失败返回 403/302，优先检查 Cookie
2. **请求频率**：Box Office Mojo 虽无严格频率限制，但建议保持 `CONCURRENCY ≤ 10` 且延迟不低于 0.5 秒
3. **IP 封禁**：若触发反爬，可引入代理池（`httpx` 支持 `proxies` 参数）或降低并发
4. **异常处理**：两个爬虫均包裹 `try/except`，单条失败不影响整体流程，失败记录将在控制台打印

## 6. 数据关系与用途

```
┌─────────────────────┐
│  crawlers/          │
│  ├── top250_crawler │──→ DoubanTop250 表 ──→ 中外评分对比图
│  └── box_office_    │──→ MovieBoxOffice 表 ──→ 票房 vs 口碑气泡图
│      crawler        │
└─────────────────────┘
         │
         ▼
    PostgreSQL (与 IMDb 数据、用户数据统一存储)
```

| 爬虫 | 输出表 | 支撑功能 |
|------|--------|----------|
| `top250_crawler.py` | `douban_top250` | 豆瓣 vs IMDb 双向柱状图对比 |
| `box_office_crawler.py` | `movie_box_office` | 票房 vs 评分散点气泡图（ROI 分析） |
