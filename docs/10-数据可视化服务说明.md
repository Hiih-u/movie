# 数据可视化服务说明

## 1. 概述

`services/analysis_service.py` 中的可视化模块，负责从 PostgreSQL 数据库中提取多维度统计数据，经过聚合、拆分、矩阵转换等处理后，返回前端图表库（如 Plotly、ECharts）可直接消费的标准数据格式。

该模块主要支撑**电影数据分析大屏**或**详情页统计面板**，提供从宏观趋势到微观对比的 9 类数据接口。

---

## 2. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| ORM 查询 | **SQLAlchemy AsyncSession** | 复用项目统一的异步数据库连接 |
| 数据处理 | **Pandas** | 频次统计、分组聚合、透视转换等操作极简 |
| 数据输出 | **原生 Python 数据结构**（dict/list） | 前端图表库通用，无需额外序列化依赖 |
| 数据来源 | **TitleBasics + TitleRatings + DoubanTop250 + MovieBoxOffice** | 多表关联，覆盖基础信息、评分、豆瓣数据、票房 |

---

## 3. 可视化接口详解

### 3.1 热门高分榜 (`get_top_movies`)

```python
async def get_top_movies(limit=10)
```

**功能**：返回评分最高的 N 部电影，用于首页热门推荐或排行榜。

**筛选条件**：
- `numVotes > 10000`：确保影片有足够的大众投票基础，避免小众高分干扰
- 按 `averageRating` 降序排列

**返回格式**：
```python
[('The Shawshank Redemption', 9.3), ('The Godfather', 9.2), ...]
```

**适用图表**：横向条形图、排行榜列表

---

### 3.2 年度发行趋势 (`get_year_stats`)

```python
async def get_year_stats(limit=20)
```

**功能**：统计最近 N 个年份的电影发行数量，展示行业产能趋势。

**处理逻辑**：
1. 过滤有效年份（非空、不超过当前年份）
2. 按 `startYear` 分组计数
3. 降序取最近 `limit` 年
4. **反转顺序**返回，使时间轴从左到右递增

**返回格式**：
```python
[(2005, 892), (2006, 945), ... (2024, 1234)]  # 已按年份升序排列
```

**适用图表**：折线图、面积图（X轴=年份，Y轴=发行数量）

---

### 3.3 数据库概览 (`get_stats_summary`)

```python
async def get_stats_summary()
```

**功能**：返回数据库的宏观统计指标，用于仪表盘顶部 KPI 卡片。

**统计项**：
- **电影总数**：`COUNT(TitleBasics.tconst)`
- **平均评分**：`AVG(TitleRatings.averageRating)`，保留 2 位小数

**返回格式**：
```python
(movie_count, avg_rating)  # 如 (850000, 6.42)
```

**适用图表**：KPI 数字卡片、仪表盘指标

---

### 3.4 题材偏好分布 (`get_genre_distribution`)

```python
async def get_genre_distribution(limit=5000)
```

**功能**：统计热门电影中各类型（genre）的出现频次，揭示用户/市场的题材偏好。

**处理逻辑**：
1. 取 `numVotes` 最高的 `limit` 部电影（默认 5000 部）
2. 将 `genres` 字段（逗号分隔多类型）拆分为独立类型标签
3. 使用 Pandas `value_counts()` 统计频次
4. 过滤掉 `\N` 无效值
5. 取 **Top 15** 主流类型

**返回格式**：
```python
[
    {'genre': 'Drama', 'count': 1842},
    {'genre': 'Comedy', 'count': 1523},
    ...
]
```

**适用图表**：**玫瑰图（Nightingale Chart）**、饼图、环形图

---

### 3.5 评分深度分析 (`get_rating_distribution_by_genre`)

```python
async def get_rating_distribution_by_genre(limit=5000)
```

**功能**：获取各主流类型下每部电影的评分分布数据，用于展示不同类型影片的质量离散程度。

**处理逻辑**：
1. 取热门电影（`numVotes` 降序 Top `limit`）
2. 每部电影按拆分后的类型标签展开为多行记录
3. 筛选 **Top 10** 高频类型（防止图表拥挤）
4. 返回每对 `(genre, rating)` 的原始列表

**返回格式**：
```python
[
    {'genre': 'Drama', 'rating': 7.8},
    {'genre': 'Drama', 'rating': 6.5},
    {'genre': 'Comedy', 'rating': 8.1},
    ...
]
```

**适用图表**：**箱线图（Box Plot）**、**小提琴图（Violin Plot）**——直观展示各类型评分的四分位数、异常值与分布密度

---

### 3.6 时空演变热力图 (`get_genre_evolution`)

```python
async def get_genre_evolution(limit=10000)
```

**功能**：分析 1980 年以来，不同类型电影在不同年代的分布热度，揭示流行题材的时空变迁。

**处理逻辑**：
1. 取 1980 年后、按 `numVotes` 降序的热门电影（Top `limit`）
2. 将年份归类为**年代**（如 1993 → 1990s）
3. 拆分类型标签
4. 按 `(decade, genre)` 分组计数
5. 筛选 **Top 10** 主流类型
6. 将结果转换为 **二维矩阵**，适配前端热力图组件

**返回格式**：
```python
(
    [1980, 1990, 2000, 2010, 2020],     # x轴：年代
    ['Action', 'Comedy', 'Drama', ...],  # y轴：类型
    [
        [45, 120, 230, 380, 520],        # Action 在各年代的计数
        [60, 150, 280, 410, 590],        # Comedy 在各年代的计数
        ...
    ]  # z轴：热力强度
)
```

**适用图表**：**热力图（Heatmap）**——X轴=年代，Y轴=类型，色深=影片数量

---

### 3.7 评分与热度散点图 (`get_scatter_data`)

```python
async def get_scatter_data(limit=2000)
```

**功能**：展示电影评分与投票人数的关系，揭示"口碑 vs 大众认知度"的分布规律。

**处理逻辑**：
1. 取 `numVotes` 降序 Top `limit` 部电影
2. 提取标题、评分、投票数、主类型（取 `genres` 第一个作为颜色分类依据）

**返回格式**：
```python
[
    {'title': 'Inception', 'rating': 8.8, 'votes': 2300000, 'genre': 'Action'},
    {'title': 'The Room', 'rating': 3.7, 'votes': 89000, 'genre': 'Drama'},
    ...
]
```

**适用图表**：**散点图（Scatter Plot）**——X轴=评分，Y轴=投票数，气泡大小=热度，颜色=主类型

---

### 3.8 中外评分对比 (`get_cultural_comparison`)

```python
async def get_cultural_comparison()
```

**功能**：对比同一部电影在豆瓣和 IMDb 两大平台的评分差异，揭示文化审美偏差。

**处理逻辑**：
1. **首选关联方式**：通过 `DoubanTop250.imdb_id` 与 `TitleRatings.tconst` 精确匹配
2. **兜底关联方式**：若 ID 关联为空，通过 `DoubanTop250.title` 与 `TitleBasics.primaryTitle` 标题模糊匹配
3. 取豆瓣 Top 250 的前 20 条

**返回格式**：
```python
[
    {'title': '肖申克的救赎', 'douban': 9.7, 'imdb': 9.3},
    {'title': '霸王别姬', 'douban': 9.6, 'imdb': 8.1},
    ...
]
```

**适用图表**：**双向柱状图**、分组柱状图——同一部电影左右两根柱子分别代表豆瓣与 IMDb 评分

> **注意**：该接口依赖爬虫正确填充 `DoubanTop250.imdb_id`，否则将降级到标题匹配，可能存在误差。

---

### 3.9 商业价值 vs 艺术口碑 (`get_roi_scatter_data`)

```python
async def get_roi_scatter_data(limit=1000)
```

**功能**：探索电影票房收入与评分质量之间的关联，识别"叫好不叫座"与"叫座不叫好"的影片。

**处理逻辑**：
1. 连接 `TitleBasics`、`TitleRatings`、`MovieBoxOffice` 三表
2. 过滤 `box_office > 0`（仅取有票房记录的影片）
3. 按票房降序取 Top `limit`
4. 取 `genres` 第一个作为主类型（气泡颜色分类）

**返回格式**：
```python
[
    {
        'title': 'Avatar',
        'genre': 'Action',
        'rating': 7.9,
        'votes': 1800000,      # 气泡大小
        'box_office': 2923706026  # X轴（票房）
    },
    ...
]
```

**适用图表**：**气泡图（Bubble Chart）**——X轴=票房，Y轴=评分，气泡大小=投票数，颜色=类型

---

## 4. 通用分类过滤

所有涉及 `MovieSummary` / `TitleBasics` 查询的接口，共享同一套 `_apply_category_filter` 逻辑：

| 分类参数 | 过滤条件 |
|----------|----------|
| `'movie'` | `titleType` ∈ `['movie', 'tvMovie']`，排除 `Documentary` |
| `'tv'` | `titleType` ∈ `['tvSeries', 'tvMiniSeries']` |
| `'anime'` | `genres` 包含 `Animation` |
| `'variety'` | `genres` 包含 `Reality-TV` / `Talk-Show` / `Game-Show` |
| `'doc'` | `genres` 包含 `Documentary` |

## 5. 数据流总览

```
PostgreSQL
    ├── TitleBasics      ──→ 年份、类型、标题、titleType
    ├── TitleRatings     ──→ averageRating、numVotes
    ├── DoubanTop250     ──→ douban_score、imdb_id（可选）
    └── MovieBoxOffice   ──→ box_office
                │
                ▼
    [analysis_service 可视化接口]
                │
                ├── get_top_movies ───────────→ 排行榜
                ├── get_year_stats ───────────→ 折线图/面积图
                ├── get_stats_summary ────────→ KPI 卡片
                ├── get_genre_distribution ─────→ 玫瑰图/饼图
                ├── get_rating_distribution_by_genre → 箱线图/小提琴图
                ├── get_genre_evolution ──────→ 热力图
                ├── get_scatter_data ─────────→ 散点图
                ├── get_cultural_comparison ──→ 双向柱状图
                └── get_roi_scatter_data ─────→ 气泡图
```

## 6. 调用示例

```python
from services.analysis_service import (
    get_top_movies, get_genre_distribution,
    get_genre_evolution, get_cultural_comparison
)

# 1. 热门榜
movies = await get_top_movies(limit=10)

# 2. 题材分布（玫瑰图）
genre_data = await get_genre_distribution(limit=5000)
# 前端 ECharts: roseType='area'

# 3. 时空演变（热力图）
years, genres, matrix = await get_genre_evolution(limit=10000)
# 前端 Plotly: px.imshow(matrix, x=years, y=genres)

# 4. 中外评分对比
comparison = await get_cultural_comparison()
# 前端: 双向柱状图，左为豆瓣，右为 IMDb
```

---

## 7. 注意事项

1. **性能考量**：`get_genre_evolution`、`get_scatter_data` 等接口取数上限较高（10000+），建议在生产环境增加数据库索引（`TitleRatings.numVotes`、`TitleBasics.startYear`）
2. **Pandas 依赖**：`get_genre_distribution`、`get_rating_distribution_by_genre`、`get_genre_evolution` 内部使用 Pandas 做聚合与透视，需确保环境已安装
3. **豆瓣数据完整性**：`get_cultural_comparison` 依赖 `DoubanTop250.imdb_id` 字段，若爬虫未回填，将降级为标题匹配，可能出现错配或空结果
4. **票房数据稀疏性**：`get_roi_scatter_data` 仅覆盖有票房记录的电影（通常为中/美商业大片），文艺片或老片可能不在统计范围内
5. **类型拆分一致性**：所有涉及 `genres` 字段的接口均按逗号 `,` 拆分，假设 IMDb 原始数据格式为 `Action,Adventure,Sci-Fi`
