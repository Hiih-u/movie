### 第六章 系统测试

系统测试是软件开发生命周期中至关重要的一环，旨在通过模拟真实的业务场景，验证系统功能是否符合需求规格说明，并评估其在性能、稳定性和安全性方面的表现。本章将详细阐述影视推荐系统（MovieRec Sys）的测试环境、测试用例设计以及最终的测试结果分析。

#### 6.1 测试环境与工具说明

为了确保测试结果的准确性与可参考性，本系统的测试工作在与开发环境高度一致的受控环境下进行。测试环境涵盖了硬件配置、软件架构以及辅助测试工具三个维度。

**6.1.1 硬件环境**
由于系统涉及千万级数据的离线处理（Spark）与大模型推理（NLP），测试主机需具备一定的并行计算能力。

* **处理器 (CPU)**: 12th Gen Intel(R) Core(TM) i7-12700H (14 核心 / 20 线程)，保证了 Spark 本地模式下多任务并发的执行效率。
* **内存 (RAM)**: 32GB DDR5，为 PostgreSQL 的海量数据缓存与 Spark 矩阵运算提供充足的物理内存，防止测试过程中出现 OOM（内存溢出）错误。
* **存储 (Disk)**: 1TB NVMe SSD，确保了千万级 TSV 原始数据导入（\COPY）以及索引构建过程中的高速 I/O 吞吐。

**6.1.2 软件环境**
系统测试建立在现代化的全栈异步技术栈之上，具体的软件版本配置如下：

* **操作系统**: Ubuntu 22.04 LTS (Linux Kernel 5.15)，提供了稳定的多进程调度与网络 I/O 性能。
* **编程语言**: Python 3.11，利用其最新的性能优化特性与 `asyncio` 异步运行时。
* **数据库管理系统**: PostgreSQL 15.x，作为核心的关系型数据底座。
* **大数据引擎**: Apache Spark 3.x 配合 Java JDK 17，用于执行分布式的 ALS 推荐算法。
* **浏览器**: Google Chrome 120.0 (或更新版本)，用于验证 NiceGUI 前端组件的渲染兼容性与 WebSocket 通信的稳定性。

**6.1.3 测试工具集**
为了实现多维度的覆盖测试，本系统引入了以下专业测试工具：

* **单元测试工具 (Pytest)**: 用于对后端核心 Service 逻辑（如 `auth_service`, `movie_service`）进行原子化功能测试。
* **接口测试工具 (Postman / FastAPI Swagger)**: 针对 FastAPI 暴露的异步路由接口进行压力测试与边界值校验。
* **前端调试工具 (Chrome DevTools)**: 监控 NiceGUI 前端与后端的 WebSocket 心跳包流量，排查大屏可视化图表加载时的延迟问题。
* **数据库监控工具 (pgAdmin 4)**: 监控大规模查询时的 SQL 执行计划（Explain Analyze）与索引命中率。
* **NLP 模型基准测试**: 利用 Hugging Face 提供的 `evaluate` 库对 `chinese_xlm_xnli` 模型在自定义测试集上的识别准确率进行量化分析。
