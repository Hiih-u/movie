### 3.2 系统总体功能模块设计

基于前述的需求分析与系统架构设计，本系统（MovieRec Sys）在功能层面进行了细粒度的垂直拆分。为了保障系统的易用性与管理的规范性，系统总体功能划分为三大核心模块：**前台用户模块**、**后台管理模块**以及独立运行的**离线计算模块**。

#### 3.2.1 前台用户模块设计

前台模块直接面向终端用户，旨在提供流畅的影视检索、个性化推荐发现以及丰富的情感交互体验。该模块主要包含以下五个子功能：

1. **用户鉴权与画像构建**
* 提供基础的注册与登录功能。在用户注册阶段，除了要求输入账号密码外，系统特别设计了“完善画像”步骤，引导用户填写性别、年龄以及职业等基础人口统计学信息。这些静态画像数据将与后期的动态行为数据（打分、收藏）相结合，为后续多维度的个性化推荐提供数据支撑。


2. **影视检索与分类浏览**
* 系统首页提供直观的全局模糊搜索框，支持用户通过输入影视名称快速定位目标作品。
* 提供精细化的分类导航栏，用户可自由在“全部”、“电影”、“电视剧”、“动漫”、“综艺”与“纪录片”等频道间无缝切换。分类查询底层通过 SQLAlchemy 构建复杂的 `WHERE` 过滤条件实现精准匹配。


3. **多策略混合推荐展示**
* **猜你喜欢**：在首页侧边栏，系统根据登录用户的身份展示专属推荐列表。该列表的生成融合了瀑布流降级策略：优先展示 Spark 离线计算的高精度推荐结果；若离线数据不足，则实时调用基于内存的 Item-CF 算法进行补充推荐；对于毫无历史交互行为的“冷启动”新用户，系统则自动降级，为其展示当前所选分类下的全局热门高分榜单。


4. **情感树洞（NLP 交互推荐）**
* 作为系统的特色功能，用户可在“情感树洞”输入框中随意倾诉当前的心理状态或口语化长句（如“今天遇到很多挫折，心情很低落”）。系统通过后端集成的 NLP 零样本分类大模型捕获其情绪意图，并弹出匹配该情绪状态（如“难过”对应“治愈系”喜剧或动画）的专属定制片单与暖心文案。


5. **用户互动与个人中心**
* **收藏与评分**：用户可在电影卡片上直接进行“红心收藏”或在弹窗中进行“1-10分”滑动打分。这些交互动作不仅会持久化到数据库，还会实时改变该用户的在线推荐权重。
* **个人中心**：提供独立的“我的收藏”和“我的评分”管理页面，用户可以分页回顾、修改或删除个人的历史交互记录。



#### 3.2.2 后台管理模块设计

后台管理模块面向拥有 `admin` 权限的系统管理员，提供全库数据的监控、分析与维护能力。主要包括以下功能：

1. **数据分析与可视化大屏**
* **监控仪表盘**：实时展示全库影视总数、全网平均分等宏观指标。管理员可在此页面监控“协同过滤推荐引擎”的加载状态，并一键手动触发算法模型的重新训练。
* **可视化大屏**：基于离线聚合数据，展示包括“ROI 商业价值气泡图”、“题材偏好玫瑰图”、“电影类型时代变迁热力图”以及“豆瓣与 IMDb 中西审美差异双向柱状图”在内的深度行业洞察报表，为平台运营提供直观的数据支撑。


2. **影视资源综合管理**
* 提供针对 `title_basics`（主数据）、`movie_summary`（前端首页缓存数据）以及 `title_episode`（剧集分集数据）的高性能分页数据表格。管理员可以对庞大的影视库进行增、删、改、查，并在修改底层数据后，通过“重建缓存”功能一键同步数据至前端展示层。


3. **演职员与剧组库管理**
* 维护千万级的演职人员名单（包含姓名、生卒年、代表作）及影视剧组绑定关系（导演、编剧配置）。该模块保障了影视底库关系网的完整性，为后续基于内容的深度推荐保留了扩展空间。


4. **用户与评分体系管理**
* 管理员有权查阅、编辑平台所有注册用户的画像信息，调整用户角色权限，并重置异常用户的密码。
* 评分管理模块允许管理员对底层全局平均分及总投票数进行微调与异常值干预，维护平台评分体系的公信力。



#### 3.2.3 离线计算与数据汇聚模块设计

为了不占用在线 Web 服务的系统资源，保障前台毫秒级的响应速度，系统将耗时的大数据计算与外部网络 I/O 任务剥离为独立的模块运行：

1. **Spark 分布式推荐引擎（离线）**
* 作为一个独立的 Python 脚本作业（Spark Job）运行。该模块利用 PySpark 框架，定期从数据库读取海量用户评分历史，通过交替最小二乘法（ALS）进行分布式模型拟合。随后，为全量用户生成长期的预测 Top-N 推荐列表，并采用批量 `Truncate` 与 `Append` 机制将千万级预测结果无缝回写至数据库的离线推荐表（`spark_recommendations`）中，供前台调用。


2. **异步数据爬虫集群**
* 包含针对 Box Office Mojo 全球票房数据的抓取脚本与针对豆瓣 Top 250 榜单的抓取脚本。
* 该模块采用 Python 原生 `asyncio` 协程机制结合 HTTPX 实现多线程并发抓取，通过动态解析 HTML DOM 结构（BeautifulSoup），高效提取缺失的票房数值与中文评分等信息，最后通过 Upsert（存在则更新，不存在则插入）策略将增量数据同步至核心数据库，丰富了平台的数据维度。
