### 3.4.1 基于交替最小二乘法 (ALS) 的离线矩阵分解

在面对系统接入的千万级 IMDb 影视数据与海量用户交互记录时，传统的基于内存的协同过滤算法（如 User-CF 或 Item-CF）会面临极其严重的“数据稀疏性（Data Sparsity）”问题。由于绝大多数用户仅对极少部分影视作品进行过评分或收藏，导致用户-物品评分矩阵中存在大量空白，这极大地限制了传统算法的推荐覆盖率与计算效率。

为了解决这一问题，本系统在离线推荐计算引擎中引入了基于模型（Model-based）的矩阵分解技术，具体采用了 Apache Spark MLlib 框架下高度优化的**交替最小二乘法（Alternating Least Squares, ALS）**。

**1. 矩阵分解的核心数学原理**

矩阵分解的基本思想是：将一个维度极高且高度稀疏的“用户-物品”评分矩阵 $R_{m \times n}$（其中 $m$ 为用户数，$n$ 为影视作品数），近似分解为两个低维且稠密的特征矩阵的乘积：


$$R \approx U \times V^T$$


其中，$U_{m \times k}$ 代表**用户隐特征矩阵**（User Latent Features），每行代表一个用户在 $k$ 维隐空间中的偏好向量；$V_{n \times k}$ 代表**物品隐特征矩阵**（Item Latent Features），每行代表一部影视作品在相同 $k$ 维隐空间中的属性向量。这里的 $k$ 远小于 $m$ 和 $n$。

通过这种降维映射，系统可以将用户对某部未观看电影的预测评分 $\hat{r}_{ui}$ 转化为两个隐特征向量的内积：


$$\hat{r}_{ui} = u_i^T v_j$$

**2. ALS 的优化目标与交替求解**

为了求解出矩阵 $U$ 和 $V$，系统需要最小化预测评分与真实已知评分之间的误差。为了防止模型在训练集上过拟合，ALS 在损失函数（Loss Function）中引入了 L2 正则化项（Tikhonov Regularization）。其目标函数定义如下：


$$L(U, V) = \sum_{(i,j) \in K} (r_{ij} - u_i^T v_j)^2 + \lambda (\sum_i ||u_i||^2 + \sum_j ||v_j||^2)$$


其中，$K$ 表示所有包含真实评分的 $(i, j)$ 集合，$\lambda$ 为正则化惩罚系数。

由于目标函数中 $U$ 和 $V$ 是相乘的关系，这是一个非凸优化问题，直接求解全局最优解极其困难。ALS 算法巧妙地采用了“交替固定”的策略将问题转化为二次凸优化问题：

* **步骤一**：首先随机初始化矩阵 $V$。
* **步骤二（固定 V 求解 U）**：将 $V$ 视为常数矩阵，此时目标函数退化为关于 $U$ 的二次函数。通过对 $u_i$ 求偏导并令其为零，可以利用标准最小二乘法独立、并行地求解出 $U$ 的最优解。
* **步骤三（固定 U 求解 V）**：同理，将刚刚求得的 $U$ 视为常数矩阵，对 $v_j$ 求偏导并求解出 $V$ 的最优解。
* **步骤四**：不断重复步骤二和步骤三，直到损失函数收敛或达到预设的最大迭代次数。

**3. ALS 在本系统中的工程落地（PySpark 实现）**

在系统的 `spark_runner.py` 离线计算作业中，ALS 算法的实施被封装为一条高效的分布式数据处理流水线：

1. **分布式数据提取与编码**：Spark 通过 JDBC 批量拉取 PostgreSQL 中的 `user_personal_ratings` 表数据。由于算法要求输入必须是数值型索引，而系统底层影视主键（如 `tt0111161`）为字符串，系统引入了 `StringIndexer` 特征转换算子，在内存中将海量字符串 ID 映射为稠密的整型 ID。
2. **模型拟合与超参数控制**：系统实例化 `pyspark.ml.recommendation.ALS` 模型。为了平衡计算精度与资源消耗，系统设定了最大迭代次数（`maxIter=10`）、正则化参数（`regParam=0.1`）以及隐特征维度（`rank=10`）。同时，配置 `coldStartStrategy="drop"` 以过滤掉模型预测过程中的 NaN 值。
3. **Top-N 推荐预测**：模型训练完成后，调用 `recommendForAllUsers(10)` 方法，Spark 会在分布式集群中为每位用户并行计算并输出预测评分最高的 10 部影视作品。
4. **逆向解码与持久化**：生成的预测结果为嵌套的数组结构。系统利用 Spark SQL 的 `explode` 算子将数组扁平化，并通过 `IndexToString` 将整型 ID 逆向还原为原始的 IMDb 字符串主键，最后通过批量覆盖（`mode="overwrite"`）的方式将千万级离线推荐结果持久化回数据库的 `spark_recommendations` 表中，供前台随时调用。
