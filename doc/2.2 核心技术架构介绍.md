### 2.2 核心技术架构介绍

本系统打破了传统 Web 推荐系统前后端分离的繁琐架构，采用了一套基于纯 Python 生态的现代化、全异步（Asynchronous）技术栈。本节将详细介绍构建该系统的核心开发框架、算法库与技术组件。

#### 2.2.1 全栈开发框架：NiceGUI 与 FastAPI

本系统的前端界面与后端路由均由 **NiceGUI** 框架驱动。

* **底层驱动**：NiceGUI 的底层是著名的现代高性能 Python Web 框架 **FastAPI**，这使得系统天然具备极高的并发处理能力和异步 I/O 特性。
* **声明式 UI 渲染**：NiceGUI 允许开发者使用 Python 代码直接声明式地编写前端 UI（内部自动映射为 Vue.js 组件），从而极大降低了前后端通信的复杂度。系统中的各类交互组件（如导航栏、数据表格、情感树洞弹窗等）均通过此方式构建。此外，该框架原生集成了对 WebSocket 的支持，使得前端状态的实时更新与后台计算任务的进度反馈变得极为平滑。

#### 2.2.2 异步对象关系映射：SQLAlchemy 2.0

在持久层数据交互方面，系统采用了 **SQLAlchemy** 作为对象关系映射（ORM）工具。

* **异步引擎**：为了与 FastAPI 的异步机制完美契合，系统使用了 SQLAlchemy 配合 `asyncpg` 驱动构建了 `AsyncSession`。这种纯异步的数据访问模式避免了在高并发场景下因数据库查询阻塞主线程的性能瓶颈。
* **数据建模**：通过 SQLAlchemy 的声明式基类（`declarative_base`），系统将复杂的 IMDb 影视关系网（如 TitleBasics, TitleRatings, TitleCrew 等）严格映射为 Python 面向对象模型，保障了业务逻辑层代码的清晰与安全。

#### 2.2.3 大数据与推荐算法计算引擎：PySpark & Scikit-learn

作为推荐系统的核心大脑，本课题采用了两套并行的计算引擎以实现混合推荐（Hybrid Recommendation）。

* **离线计算（Apache Spark）**：面对百万级别的用户与千万级别的影视交互数据，单机计算往往无能为力。系统引入了 Apache Spark，并通过 **PySpark** 的机器学习库（MLlib）实现了交替最小二乘法（ALS）模型。ALS 算法通过分布式计算进行矩阵分解，能够高效地为全量用户生成离线 Top-N 推荐列表并回写至数据库，具备极强的大规模数据处理能力。
* **实时计算（Scikit-learn）**：为了捕捉用户短期的即时兴趣转移，系统使用轻量级的 **Scikit-learn** 库计算基于物品的余弦相似度矩阵（Item-Based CF）。这种基于内存的计算方式能够在用户触发“评分”或“收藏”动作时，于毫秒级生成即时推荐反馈。

#### 2.2.4 自然语言处理与情感计算：Transformers 大模型

本系统的特色创新点在于基于认知智能的“情感推荐”，该功能主要依赖于 Hugging Face 提供的 **Transformers** 库。

* **零样本分类（Zero-Shot Classification）**：系统在后端集成了开源的 `chinese_xlm_xnli` 预训练模型。通过构建零样本分类流水线（Pipeline），系统能够直接理解用户输入的无结构化自然语言文本，并计算其与系统预设情感类别（如“开心”、“难过”、“愤怒”等）的匹配置信度，从而实现高精度的情感识别与意图捕获。

#### 2.2.5 数据可视化与分析生态：Pandas & Plotly

后台数据大屏的渲染直接体现了系统的商业分析价值。

* **数据清洗与聚合（Pandas）**：在生成复杂图表前，系统广泛使用 **Pandas** 库对数据库查询出的原始散点数据进行清洗、透视与多维聚合（如按年代归类、按题材统计）。
* **动态图表渲染（Plotly）**：相较于传统的静态图表库（如 Matplotlib），系统采用了工业级的 **Plotly**（包括 Plotly Express 与 Graph Objects）。Plotly 能够直接生成具备强交互性（支持悬停、缩放、图例筛选）的动态 HTML 图表组件，完美嵌入 NiceGUI 前端框架中，实现了极富科技感的可视化大屏展示。
