### 4.2 逻辑结构设计

逻辑结构设计是将前一节构建的概念模型（E-R 图）转化为具体的数据库表结构的过程。鉴于本系统采用了 PostgreSQL 作为底层数据库，逻辑设计中充分利用了 PostgreSQL 对复杂数据类型（如 `ARRAY` 数组）和高并发存取的原生支持。以下详细罗列了系统中各核心数据表的字段定义、数据类型及约束规范。

#### 4.2.1 用户与交互数据表

该部分表主要负责维系系统的账号安全体系以及沉淀用户产生的高价值交互行为数据，是实现个性化推荐的数据源泉。

**1. 用户基础信息表（`users`）**
用于存储平台所有注册用户及管理员的核心画像与鉴权信息。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `id` | Integer | Primary Key, Auto Increment | 用户唯一标识自增主键 |
| `username` | String(50) | Unique, Not Null | 登录用户名，需保证全局唯一 |
| `hashed_password` | String(255) | Not Null | 经 bcrypt 算法加密后的哈希密码 |
| `role` | String(20) | Default 'user' | 角色权限标识（如 'user', 'admin'） |
| `gender` | String(10) | Nullable | 用户填写的性别画像 |
| `age_group` | String(20) | Nullable | 用户填写的年龄段画像 |
| `created_at` | DateTime | Default CURRENT_TIMESTAMP | 账号注册时间 |

**2. 用户个人评分表（`user_personal_ratings`）**
记录用户对具体影视作品的“1-10分”滑动打分明细。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `id` | Integer | Primary Key, Auto Increment | 评分记录自增主键 |
| `user_id` | Integer | Foreign Key (`users.id`), Index | 关联的用户 ID |
| `tconst` | String(15) | Foreign Key (`title_basics.tconst`), Index | 关联的 IMDb 影视编号 |
| `rating` | Float | Check (0 <= rating <= 10) | 用户提交的具体评分值 |
| `created_at` | DateTime | Default CURRENT_TIMESTAMP | 交互发生的时间 |

**3. 用户收藏记录表（`user_favorites`）**
记录用户的“红心收藏”动作，在协同过滤模型中作为强意愿权重。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `id` | Integer | Primary Key, Auto Increment | 收藏记录自增主键 |
| `user_id` | Integer | Foreign Key (`users.id`), Index | 关联的用户 ID |
| `tconst` | String(15) | Foreign Key (`title_basics.tconst`), Index | 关联的 IMDb 影视编号 |
| `created_at` | DateTime | Default CURRENT_TIMESTAMP | 收藏动作触发的时间 |

#### 4.2.2 影视核心基础表

该部分表承载了 IMDb 千万级的影视底库数据。为了兼顾数据的完整性与前端的高效查询，系统设计了规范化的主表与反范式化的缓存宽表。

**1. 影视基础主表（`title_basics`）**
映射自 IMDb 官方 `title.basics.tsv` 数据集，存储影视作品的核心客观属性。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `tconst` | String(15) | Primary Key | 影视全局唯一编号（如 `tt0111161`） |
| `titleType` | String(20) | Index | 作品类型（movie, tvSeries, short 等） |
| `primaryTitle` | String(500) | Not Null | 主要展示标题 |
| `originalTitle` | String(500) | Nullable | 原始语言标题 |
| `isAdult` | Boolean | Default False | 是否为成人级限制内容 |
| `startYear` | Integer | Index, Nullable | 首映年份 |
| `runtimeMinutes` | Integer | Nullable | 播放时长（分钟） |
| `genres` | ARRAY(String) | Nullable | 题材分类集合（PostgreSQL 数组类型，支持多标签） |
| `poster_url` | String(255) | Nullable | 通过 TMDB API 动态抓取的海报链接 |

**2. 前端展示缓存宽表（`movie_summary`）**
由于在首页进行多表联查（JOIN 主表、评分表、类型表等）并进行排序会极大消耗数据库性能，系统设计了该缓存表用于前台大批量数据的分页与检索。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `tconst` | String(15) | Primary Key, Foreign Key | 关联的主键 ID |
| `primaryTitle` | String(500) | Index | 冗余的电影标题（用于模糊搜索） |
| `titleType` | String(20) | Index | 冗余的类型标识 |
| `startYear` | Integer | Index | 冗余的年份 |
| `genres` | ARRAY(String) | GIN Index | 冗余的题材标签，配置 GIN 索引以加速数组检索 |
| `averageRating` | Float | Index | 从 `title_ratings` 同步的全网平均分 |
| `numVotes` | Integer | Index | 从 `title_ratings` 同步的总投票数 |
| `poster_url` | String(255) | Nullable | 冗余的海报地址 |

#### 4.2.3 演职人员与剧组关联表

用于构建庞大的影视关系图谱，支持基于演员、导演等维度的深度查询与扩展推荐。

**1. 演职人员主表（`name_basics`）**
映射自 IMDb 官方 `name.basics.tsv` 数据集。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `nconst` | String(15) | Primary Key | 演职人员唯一编号（如 `nm0000001`） |
| `primaryName` | String(255) | Not Null | 姓名 |
| `birthYear` | Integer | Nullable | 出生年份 |
| `deathYear` | Integer | Nullable | 死亡年份 |
| `primaryProfession`| ARRAY(String) | Nullable | 核心职业列表（如 actor, producer） |
| `knownForTitles` | ARRAY(String) | Nullable | 代表作影片 ID 数组 |

**2. 剧组人员关系表（`title_principals` / `title_crew`）**
记录具体某部影片的演职人员构成，是 `title_basics` 与 `name_basics` 之间的多对多中间表。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `id` | Integer | Primary Key, Auto Increment | 关系流水号 |
| `tconst` | String(15) | Foreign Key, Index | 关联的影视 ID |
| `nconst` | String(15) | Foreign Key, Index | 关联的演职人员 ID |
| `ordering` | Integer | Nullable | 在职员表中的排序权重 |
| `category` | String(50) | Index | 职位分类（如 director, actor, writer） |
| `job` | String(255) | Nullable | 具体的职务细分 |
| `characters` | String(500) | Nullable | 饰演的角色名称（通常为 JSON 字符串形式） |

#### 4.2.4 扩展数据表

为了增强系统数据的丰富度与商业分析价值，系统接入了外部爬取的数据与内部离线计算产生的数据。

**1. 豆瓣 Top250 榜单表（`douban_top250`）**
存储由异步爬虫抓取的中文互联网权威榜单数据，用于中西文化审美差异对比大屏。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `id` | Integer | Primary Key, Auto Increment | 记录主键 |
| `rank` | Integer | Unique | 豆瓣历史排名（1-250） |
| `title` | String(255) | Not Null | 豆瓣中文译名 |
| `douban_rating` | Float | Not Null | 豆瓣网友评分 |
| `votes` | Integer | Not Null | 豆瓣打分人数 |
| `imdb_id` | String(15) | Foreign Key (`title_basics.tconst`) | 手动/正则对齐的底层 IMDb 编号 |

**2. 全球票房数据表（`movie_box_office`）**
存储抓取自 Box Office Mojo 的影片商业价值数据，支撑 ROI 气泡图的绘制。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `tconst` | String(15) | Primary Key, Foreign Key | 关联的底层影视编号 |
| `worldwide_box_office`| BigInteger | Nullable | 全球总票房（美元） |
| `domestic_box_office` | BigInteger | Nullable | 北美本土票房（美元） |
| `international_box_office`| BigInteger | Nullable | 国际票房（美元） |

**3. 离线模型推荐结果表（`spark_recommendations`）**
专门用于持久化存放由 Apache Spark 集群通过 ALS 算法计算生成的离线全局推荐结果。
| 字段名称 | 数据类型 | 约束条件 | 描述说明 |
| :--- | :--- | :--- | :--- |
| `id` | Integer | Primary Key, Auto Increment | 记录主键 |
| `user_id` | Integer | Foreign Key (`users.id`), Index | 目标用户 ID |
| `tconst` | String(15) | Foreign Key (`title_basics.tconst`), Index | 被推荐的影视 ID |
| `predicted_rating` | Float | Not Null | ALS 模型计算出的预测偏好得分 |
| `created_at` | DateTime | Default CURRENT_TIMESTAMP | 离线计算批次的时间戳，用于定期清理 |
