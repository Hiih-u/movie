### 4.2 数据库逻辑结构设计

本系统使用 PostgreSQL 数据库来存储所有的影视资料和用户信息。为了让千万级的数据跑得快，我把原本杂乱的大文件拆分成了这些规整的数据表。下面是系统中所有表的具体设计方案。

#### 1. 影视基础信息表 (title_basics)

这张表是系统的核心，存的是电影和剧集的最基本资料。由于原始数据很大，我给编号字段加了索引，这样搜片子的速度会快很多。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| tconst | String | 否 | 是 | 否 | 影视唯一编号（如 tt0000001） |
| titleType | String | 是 | 否 | 否 | 作品类型（电影、电视剧等） |
| primaryTitle | String | 否 | 否 | 否 | 影视主要标题 |
| originalTitle | String | 是 | 否 | 否 | 原始标题 |
| isAdult | Integer | 否 | 否 | 否 | 是否为成人内容 |
| startYear | Integer | 是 | 否 | 否 | 开始或上映年份 |
| endYear | Integer | 是 | 否 | 否 | 完结年份 |
| runtimeMinutes | Integer | 是 | 否 | 否 | 播放时长（分钟） |
| genres | String | 是 | 否 | 否 | 题材分类 |
| poster_path | String | 是 | 否 | 否 | 海报图片路径 |
| backdrop_path | String | 是 | 否 | 否 | 背景大图路径 |
| overview | String | 是 | 否 | 否 | 剧情中文简介 |
| tmdb_id | String | 是 | 否 | 否 | 外部 TMDB 库的 ID |

#### 2. 影视评分表 (title_ratings)

这张表专门存全球观众对影视作品的评价情况。它通过编号和主表连在一起，方便我们看哪部片子口碑最好。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| tconst | String | 否 | 是 | 是 | 关联影视基础表的编号 |
| averageRating | Float | 是 | 否 | 否 | 平均得分（1-10分） |
| numVotes | Integer | 是 | 否 | 否 | 总打分人数 |

#### 3. 用户基础信息表 (users)

这张表存的是注册用户和管理员的信息。为了保护隐私，密码都经过了加密处理。我在实习期间发现，加上性别和年龄段字段能让推荐算法算得更准。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| id | Integer | 否 | 是 | 否 | 用户唯一自增 ID |
| username | String | 否 | 否 | 否 | 登录用户名（唯一） |
| hashed_password | String | 否 | 否 | 否 | 加密后的密码 |
| role | String | 否 | 否 | 否 | 角色权限（管理员或普通用户） |
| gender | String | 是 | 否 | 否 | 性别 |
| age | Integer | 是 | 否 | 否 | 年龄 |
| occupation | String | 是 | 否 | 否 | 职业 |

#### 4. 演职人员基础表 (name_basics)

系统通过这张表来展示电影背后的主创团队。它记下了演员和导演的名字以及他们的代表作。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| nconst | String | 否 | 是 | 否 | 人员唯一编号 |
| primaryName | String | 是 | 否 | 否 | 人员姓名 |
| birthYear | Integer | 是 | 否 | 否 | 出生年份 |
| deathYear | Integer | 是 | 否 | 否 | 去世年份 |
| primaryProfession | String | 是 | 否 | 否 | 主要职业 |
| knownForTitles | String | 是 | 否 | 否 | 知名代表作品 |

#### 5. 剧组人员关联表 (title_crew)

这张表把电影和导演、编剧对应起来。由于一部电影可能有多个导演，我把这些 ID 存成了文字列表。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| tconst | String | 否 | 是 | 是 | 关联影视基础表的编号 |
| directors | String | 是 | 否 | 否 | 导演编号列表 |
| writers | String | 是 | 否 | 否 | 编剧编号列表 |

#### 6. 剧集分集信息表 (title_episode)

如果是电视剧，系统会用这张表来管理每一集的内容。它能帮我们快速找出一整季里所有的单集。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| tconst | String | 否 | 是 | 否 | 这一集本身的编号 |
| parentTconst | String | 是 | 否 | 是 | 所属剧集的总编号 |
| seasonNumber | Integer | 是 | 否 | 否 | 第几季 |
| episodeNumber | Integer | 是 | 否 | 否 | 第几集 |

#### 7. 首页高性能缓存表 (movie_summary)

这是为了让系统跑得飞快而专门设计的一张表。我把电影基本资料和评分合在了一起，这样首页加载时就不用去到处找数据了。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| tconst | String | 否 | 是 | 否 | 影视唯一编号 |
| titleType | String | 是 | 否 | 否 | 作品类型 |
| primaryTitle | String | 是 | 否 | 否 | 主要标题 |
| startYear | Integer | 是 | 否 | 否 | 上映年份 |
| runtimeMinutes | Integer | 是 | 否 | 否 | 播放时长 |
| genres | String | 是 | 否 | 否 | 题材分类 |
| averageRating | Float | 是 | 否 | 否 | 平均得分 |
| numVotes | Integer | 是 | 否 | 否 | 总打分人数 |
| poster_path | String | 是 | 否 | 否 | 缓存的海报路径 |

#### 8. 用户收藏关联表 (user_favorites)

这张表记录了用户点过红心的片子。我给用户 ID 和片子编号加了联合唯一限制，这样就能在数据库层面防止重复收藏。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| id | Integer | 否 | 是 | 否 | 收藏记录 ID |
| user_id | Integer | 否 | 否 | 是 | 关联用户表的 ID |
| tconst | String | 否 | 否 | 是 | 关联影视基础表的编号 |
| created_at | DateTime | 否 | 否 | 否 | 收藏时间 |

#### 9. 用户个人评分表 (user_personal_ratings)

这张表存的是用户在平台里真实打的分数。这些分数是推荐算法最重要的参考资料。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| id | Integer | 否 | 是 | 否 | 评分记录 ID |
| user_id | Integer | 否 | 否 | 是 | 关联用户表的 ID |
| tconst | String | 否 | 否 | 是 | 关联影视基础表的编号 |
| rating | Float | 否 | 否 | 否 | 用户给出的评分（1-10分） |
| created_at | DateTime | 否 | 否 | 否 | 打分时间 |

#### 10. Spark 离线推荐结果表 (spark_recommendations)

这是用来存 Spark 算出来的推荐结果。系统定期在后台跑完算法后，会把每个用户可能喜欢的电影和预测分数存进这里。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| id | Integer | 否 | 是 | 否 | 记录 ID |
| user_id | Integer | 否 | 否 | 否 | 被推荐的用户 ID |
| tconst | String | 否 | 否 | 否 | 推荐的影片编号 |
| score | Float | 否 | 否 | 否 | 预测的评分分值 |
| algorithm | String | 否 | 否 | 否 | 使用的算法名称（如 ALS） |

#### 11. 豆瓣 Top 250 数据表 (douban_top250)

为了丰富内容，系统还爬取了豆瓣的权威榜单。这张表记下了豆瓣的排名和中文名字，方便和国外的数据做对比。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| id | Integer | 否 | 是 | 否 | 记录 ID |
| rank | Integer | 否 | 否 | 否 | 豆瓣实时排名（1-250） |
| title | String | 是 | 否 | 否 | 影视中文名称 |
| douban_id | String | 否 | 否 | 否 | 豆瓣站内的唯一 ID |
| imdb_id | String | 是 | 否 | 否 | 对应的 IMDb 编号 |
| douban_score | Float | 是 | 否 | 否 | 豆瓣网友评分 |
| created_at | DateTime | 否 | 否 | 否 | 爬取时间 |

#### 12. 电影全球票房表 (movie_box_office)

这张表专门用来存片子的吸金能力。我用了很大的数字类型来存票房金额，防止数字太长存不下。

| 字段名 | 数据类型及长度 | 空值 | 主键 | 外键 | 字段说明 |
| --- | --- | --- | --- | --- | --- |
| id | Integer | 否 | 是 | 否 | 记录 ID |
| tconst | String | 否 | 否 | 是 | 关联影视基础表的编号 |
| box_office | BigInteger | 是 | 否 | 否 | 全球累计票房（美元） |
| rank | Integer | 是 | 否 | 否 | 票房榜单排名 |
| updated_at | DateTime | 否 | 否 | 否 | 数据最后更新时间 |
