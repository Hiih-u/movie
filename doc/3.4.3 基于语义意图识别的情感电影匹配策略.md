### 3.4.3 基于语义意图识别的情感电影匹配策略

在传统的影视推荐体系中，用户的检索意图通常需要被生硬地转化为结构化的“分类标签”或“精确的影片名称”。然而，用户的观影动机往往与其实时的心理状态和情绪诉求密切相关。为了打破传统关键字检索的局限性，本系统创新性地设计了“情感树洞”模块，并提出了一套**基于语义意图识别的情感电影匹配策略**。该策略将前沿的自然语言处理技术与心理学启发式规则相结合，实现了从“非结构化文本”到“个性化观影方案”的智能映射。

**1. 基于零样本分类的语义意图捕获**

情感匹配策略的第一步是精准理解用户口语化表达背后的真实情绪。系统后端并未采用传统的基于关键词匹配或字典正则提取的笨重方法，而是引入了 Hugging Face 生态中的 `chinese_xlm_xnli` 预训练大语言模型。

当用户输入一段随意的自然语言（例如：“今天被老板骂了，感觉很受挫，想看点能让我哭出来的或者发泄一下的”）后，系统利用**零样本分类（Zero-Shot Classification）**流水线处理该文本。

* **候选标签空间**：系统预先定义了一个覆盖人类主要观影情绪维度的标签集合，包括“开心”、“难过”、“愤怒”、“疲惫”、“害怕”、“刺激”、“平静”与“烧脑”。
* **概率推断**：模型计算输入文本与上述每一个情感标签的逻辑蕴含概率（Entailment Probability）。
* **阈值过滤（Confidence Threshold）**：为了保证意图识别的准确性，系统设定了严格的置信度阈值（通常设定为 $0.3$）。只有当最高匹配标签的概率得分大于该阈值时，系统才认定捕捉到了有效的情绪意图；否则，系统会通过友好的前端提示，引导用户提供更明确的语义描述。

**2. 心理学启发的场景映射引擎**

在成功提取出用户的核心情感标签（如识别出“难过”或“疲惫”）后，系统并不会简单地去数据库搜索带有“难过”标签的电影。相反，系统内置了一个名为 `MOOD_SCENARIOS` 的启发式映射引擎。该引擎借鉴了情感调节理论（Emotion Regulation Theory），将特定的负面或正面情绪映射至能够产生“治愈”、“共鸣”或“发泄”效果的影视属性约束上。

该引擎的具体映射规则包涵三个核心维度：**题材圈定（Genres）**、**质量底线（Min Rating）**以及**排序策略（Sort Order）**。

* **情绪抚慰策略（如“难过”、“疲惫”）**：当识别到用户处于低落或疲劳状态时，系统将其映射至“喜剧（Comedy）”、“爱情（Romance）”、“动画（Animation）”或“纪录片（Documentary）”等舒缓题材。同时，为了确保影片具备真正的治愈力量，系统强制附加了极高的质量约束（如 `min_rating >= 8.0`），并按评分降序排列，确保推荐给用户的是公认的优质佳作。
* **情绪宣泄策略（如“愤怒”、“刺激”）**：当用户表达出愤怒或寻求感官刺激时，映射引擎会立即转向“动作（Action）”、“惊悚（Thriller）”、“犯罪（Crime）”或“科幻（Sci-Fi）”题材。在此场景下，系统通常采用随机排序策略（Random），为用户带来充满盲盒感与肾上腺素的观影体验。
* **认知挑战策略（如“烧脑”）**：针对寻求智力刺激的用户，系统精准圈定“悬疑（Mystery）”、“科幻（Sci-Fi）”类目，同样辅以高分底线约束，满足用户深度解谜的诉求。

**3. 动态 ORM 寻址与结果分发**

确定了具体的映射策略后，系统需要将这些抽象的规则转化为底层数据库可以执行的 SQL 查询。
在业务逻辑层，系统利用 SQLAlchemy 的高级 ORM 特性构建动态查询树：

* **数组交集匹配**：针对 PostgreSQL 中存储为数组格式的题材字段（`genres`），系统利用特定方言操作符（如 `.op('&&')`）进行高效的数组重叠判定，快速过滤出符合策略题材库的候选影片。
* **多维条件拼接**：将题材过滤、评分下限（`averageRating >= min_rating`）、以及有效投票数（如过滤掉投票数极低的冷门电影以防刷分）等条件动态拼接。
* **结果包装**：查询结果生成后，系统不仅返回电影卡片数据，还会根据捕获的情绪标签附加一段定制化的“暖心文案”（如“生活偶尔会下雨，但这些电影会为你撑伞”），最终通过 WebSocket 实时推送到前端的交互弹窗中。

通过这一整套意图识别与映射策略，本推荐系统赋予了冰冷的数据以“同理心”，极大地丰富了人机交互的维度，实现了真正意义上的情感智能推荐。
