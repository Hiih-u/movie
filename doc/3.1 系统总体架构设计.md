### 第三章 系统需求分析与总体设计

#### 3.1 系统总体架构设计

为了满足千万级海量影视数据的存储、高并发的用户访问以及复杂的混合推荐算法与 NLP 模型推理需求，本系统（MovieRec Sys）在架构设计上摒弃了传统的单体同步架构，转而采用基于现代化全栈异步生态的分层架构体系。

系统总体架构遵循“高内聚、低耦合”的软件工程原则，自上而下划分为四个核心层次：**前端展示层**、**后端业务逻辑层**、**模型计算层**以及**数据持久层**。各层之间通过清晰的接口规范与数据模型进行交互，确保了系统的高可扩展性与易维护性。

**3.1.1 前端展示层（Presentation Layer）**
前端展示层是系统与用户直接交互的视窗，承担着数据可视化展示与用户操作指令下发的任务。本系统采用前后端同构理念的 **NiceGUI** 框架构建。

* **用户服务终端**：面向普通用户，提供响应式的 Web 界面。包含影片分类导航、电影详情弹窗、个人收藏与评分管理页面，以及基于自然语言输入触发的“情感树洞”交互模块。
* **后台管理与数据大屏**：面向系统管理员，提供基于 AgGrid 的高性能数据表格以实现影视、用户、评分、演职员等多维度的 CRUD（增删改查）操作。同时，集成 Plotly 动态图表引擎，通过 `iframe` 挂载离线生成的交互式 HTML 图表，呈现全域数据分析大屏（如 ROI 气泡图、题材演变热力图）。
* **双向通信机制**：前端 UI 的事件（如点击、输入）直接绑定后端的 Python 回调函数，框架底层自动通过 WebSocket 维护前后端状态同步，实现无刷新的局部组件更新。

**3.1.2 后端业务逻辑层（Business Logic Layer）**
业务逻辑层是系统的控制中枢，基于 **FastAPI** 异步 Web 框架构建。该层向下调用模型与数据服务，向上为前端提供数据支撑。按照领域驱动设计（DDD）的思想，业务逻辑被拆解为多个高内聚的 Service 模块：

* **基础业务服务（Services）**：包括负责鉴权与状态缓存的 `auth_service.py`、负责影视基础信息检索与 TMDB 外部 API 协同更新的 `movie_service.py` 和 `tmdb_service.py`、处理用户交互（如收藏、打分原子操作）的 `interaction_service.py` 等。
* **并发控制与路由拦截**：依托 Python 的 `async/await` 协程机制处理所有网络 I/O。在路由层面，通过 `check_admin_access` 等守卫函数统一拦截越权访问，确保系统安全性。

**3.1.3 模型计算层（Model Computation Layer）**
模型计算层是本系统的核心技术壁垒，汇聚了大数据分布式计算与现代人工智能算法，主要分为三个子模块：

* **离线推荐计算引擎（Spark ALS）**：作为一个独立于 Web 服务的子系统运行。它依托 Apache Spark 大数据框架，利用 PySpark 读取全量用户评分数据，通过交替最小二乘法（ALS）进行高维矩阵分解计算，生成全局候选的 Top-N 推荐列表并持久化至数据库，以此保障推荐的广度与深度。
* **实时在线推荐引擎（Item-Based CF）**：集成在 Web 后端的 `recommendation_service.py` 中。在系统启动或管理员手动触发时，利用 Pandas 和 Scikit-learn 在内存中构建并计算基于物品的余弦相似度矩阵。当接收到前台请求时，结合用户当前的评分与收藏权重（如收藏计 10 分），进行毫秒级的线性加权排序运算，输出具有时效性的个性化推荐。
* **情感语义推理引擎（NLP Transformers）**：部署在 `analysis_service.py` 中。加载 Hugging Face 的 `chinese_xlm_xnli` 预训练语言模型，通过零样本分类（Zero-Shot Classification）Pipeline 接收前端传来的自然语言长文本，输出情绪概率分布（如感知“难过”、“愤怒”等意图），并以此映射过滤底层数据库中的题材与高分影片，实现情感共鸣推荐。

**3.1.4 数据持久层（Data Persistence Layer）**
数据持久层负责系统所有结构化与非结构化数据的安全存储与高效检索。系统选用了高可用的 **PostgreSQL** 关系型数据库，并通过 **SQLAlchemy 2.0** ORM 进行统一的面向对象建模（映射如 `TitleBasics`, `TitleRatings`, `User` 等实体表）。

* **双驱动适配策略**：针对不同业务场景的 I/O 特性，系统在持久层配置了双引擎。Web 后端业务采用 `asyncpg` 异步驱动进行无阻塞的高频短连接查询（如分页加载、状态读取）；而离线 Spark 推荐引擎则采用 `psycopg2` 同步驱动配合 JDBC 接口，以保障海量历史数据批量吞吐的稳定性。
* **异构数据汇聚**：该层不仅容纳了数百万级的 IMDb 离线源数据，还统一管理着动态爬取的豆瓣排行数据、Box Office 票房数据，以及系统运行中产生的用户行为日志、推荐结果缓存与 TMDB 海报链接，构成了系统坚实的知识底座。
