### 第二章 相关技术与理论基础

本章主要介绍支撑本影视推荐系统（MovieRec Sys）开发与运行的核心技术栈与理论基础。系统整体采用了现代化的全栈异步架构，融合了大数据分布式计算与自然语言处理技术。本节首先详细阐述系统开发过程中所采用的编程语言、数据库以及版本控制等基础系统工具。

#### 2.1 开发环境与系统工具

为了保证系统在处理千万级影视数据和高并发请求时的高效性与稳定性，本课题在开发环境和底层工具的选择上进行了严谨的考量，主要采用了 Python 3.11、PostgreSQL、Java JDK 17 以及 Git 等成熟的技术栈。

**2.1.1 核心编程语言：Python 3.11**
本系统的主力开发语言为 Python 3.11。Python 作为当前人工智能与大数据分析领域的首选语言，拥有极其丰富的第三方生态库。

* **性能与异步优势**：Python 3.11 在解释器性能上进行了大幅优化，执行速度相较于早期版本有显著提升。更重要的是，系统重度依赖 Python 的 `asyncio` 异步编程特性。无论是后端的 Web 服务处理、数据库的并发读写，还是外部 API（如 TMDB、豆瓣爬虫）的非阻塞请求，Python 3.11 的异步机制都为系统的高吞吐量提供了底层语言保障。
* **生态兼容性**：系统涉及的机器学习（Scikit-learn）、深度学习框架（PyTorch、Transformers）、大数据计算（PySpark）以及数据处理（Pandas）均需要依赖 Python 环境。Python 3.11 提供了完美的生态兼容性，使得前后端逻辑与 AI 算法能够无缝融合在一个工程项目中。

**2.1.2 核心数据库引擎：PostgreSQL**
在面临海量数据（如 IMDb 千万级别的影视条目、用户评分及演职员关系网）的存储与检索需求时，传统的小型关系型数据库往往会遇到性能瓶颈。本系统最终选用了开源界功能最强大的关系型数据库 PostgreSQL 作为数据底座。

* **海量数据处理能力**：PostgreSQL 对复杂查询、大规模数据表的 JOIN 操作以及并发写入有着极佳的支持，能够轻松承载系统初始化时导入的数百万条 IMDb 离线测试数据。
* **双引擎驱动架构**：为了最大化发挥数据库性能，系统在数据持久层设计了“双驱动”策略。在 Web 前后台服务中，采用 `asyncpg` 异步驱动配合 SQLAlchemy 2.0 实现完全非阻塞的数据库 I/O，保证高并发下的接口响应速度；而在执行离线 Spark 推荐算法时，则采用更稳定成熟的 `psycopg2` 同步驱动配合 JDBC 进行海量评分矩阵的批量读取与推荐结果的回写。

**2.1.3 运行环境支撑：Java JDK 17**
尽管系统主体使用 Python 开发，但混合推荐架构中的“离线推荐引擎”重度依赖于 Apache Spark 大数据框架。

* PySpark 本质上是通过 Py4J 桥接 Python 与 JVM（Java 虚拟机）的混合计算框架。由于最新版本的 Apache Spark 已经弃用了对旧版 Java 的支持，因此系统必须在宿主机（推荐 Linux 环境）中配置 Java JDK 17。JDK 17 作为长期支持版本（LTS），在内存管理（如 ZGC 垃圾回收器）和计算性能上更为优秀，能够确保 Spark 在进行千万级评分矩阵 ALS 分解时不会出现内存溢出（OOM）等稳定性问题。

**2.1.4 版本控制与依赖管理：Git & Git LFS**
在项目开发周期中，为了保障代码的可维护性与团队协作效率，系统全面引入了 Git 进行版本控制。

* **常规代码管理**：通过构建合理的 `.gitignore` 文件，有效隔离了虚拟环境（`venv`）、本地缓存（`.nicegui`）、临时数据（`__pycache__`）与核心源代码，确保代码仓库的轻量与整洁。
* **Git LFS (Large File Storage)**：本系统的“情感交互模块”需要集成基于 Hugging Face 的自然语言处理大模型（`chinese_xlm_xnli`）。由于大语言模型的预训练权重文件（如 `pytorch_model.bin`）体积巨大，系统利用 Git LFS 技术对这类二进制大文件进行专门管理与追踪，从而避免了传统 Git 仓库在处理超大文件时出现的膨胀与卡顿问题。
