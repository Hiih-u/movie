### 5.4 管理员后台功能实现

管理员后台是影视推荐系统（MovieRec Sys）的运营决策与资源管控中枢。该模块基于 NiceGUI 的单页应用（SPA）架构构建，为拥有 `admin` 权限的用户提供了高效率的底库维护工具与全视角的商业分析大屏。

#### 5.4.1 信息库 CRUD（增删改查）管理

系统针对影视底库中的核心实体（影视、人员、剧集、评分等）设计了统一且高效的管理界面。由于 IMDb 基础数据规模庞大，后台管理模块在工程实现上重点解决了海量数据的分页加载、实时检索与原子化编辑问题。

**1. 基于 AgGrid 的高性能数据表格实现**
在影视管理（`movie_management.py`）、人员管理（`person_management.py`）等模块中，系统集成了工业级的 **AgGrid** 插件。

* **分页与状态管理**：系统在前端通过 `page_state` 维护当前的页码与每页条数，配合后端的分页查询服务（如 `get_movies_paginated`），实现了千万级数据的分段加载，确保了管理界面在处理海量记录时依然保持秒级的响应速度。
* **模糊搜索与过滤**：工具栏配置了带回车触发功能的搜索框，支持通过影片编号（tconst）或名称进行全局检索。系统在搜索时会自动重置分页状态，并通过 UI 转圈组件（`ui.spinner`）提供直观的加载反馈。

**2. 统一的 CRUD 业务逻辑封装**
系统为每种资源类型构建了结构一致的交互流水线：

* **新增与编辑（Dialog-based）**：采用模态弹窗（`ui.dialog`）承载编辑表单。系统通过检测传入数据是否为空来复用同一套 UI 逻辑：编辑模式下，主键 ID（如 `tconst`）会被设为不可编辑（`id_input.disable()`），并回显原始数据；新增模式下则允许用户手动输入 ID 或选择作品类型。
* **影视管理特色：重建缓存**：在 `movie_management.py` 中，系统特别设计了“重建缓存”按钮。当管理员修改了底层的 `title_basics` 主表数据后，点击此按钮可触发 `movie_service.refresh_movie_summary`。该操作会将更新后的关键元数据（海报、评分等）重新同步至前端宽表 `movie_summary` 中，确保前台用户能实时看到最新的修改结果。
* **多维资源管控**：管理后台完整覆盖了影视作品、演职人员、剧集分集（`episode_management.py`）以及全局评分（`rating_management.py`）的增删改查。管理员可以对异常的全局评分（`averageRating`）和投票数进行物理修正，以维护平台的数据公信力。

#### 5.4.2 基于 Plotly 的离线数据大屏构建

为了充分挖掘千万级 IMDb 数据的行业洞察价值，系统设计了名为“影视全域数据分析”的统计大屏。该模块在架构上采用了**“离线聚合生成 + 静态 HTML 托管”**的优化策略，有效规避了海量数据实时计算导致的性能阻塞。

**1. 离线数据聚合管道（Data Pipeline）**
在 `init/generate_charts.py` 中，系统通过异步调度执行全量统计脚本。

* **数据清洗与分析（Pandas）**：系统首先从数据库中提取千万级原始数据，利用 Pandas 进行多维聚合。例如，在生成“时代变迁热力图”时，脚本将年份划分为年代区间（如 1990s），统计各题材在不同年代的产量频次，生成透视矩阵。
* **图表引擎（Plotly）**：基于聚合后的结构化数据，系统调用 Plotly Express 或 Graph Objects 生成交互式图表，并将其导出为独立的 `.html` 文件，统一存储在 `/static/charts/` 目录下。

**2. 核心可视化维度与商业逻辑**
大屏整合了六大高维度可视化看板，为影视运营提供科学支撑：

* **ROI 商业价值气泡图（ROI Bubble Chart）**：以全球票房（Box Office）为 X 轴（对数坐标）、IMDb 评分为 Y 轴、投票热度为气泡大小。该图表通过红色虚线框特别标注了“票房 > 1亿美元且评分 > 8.5”的**神作区**，直观揭示了影片在商业表现与艺术口碑之间的分布规律。
* **题材偏好玫瑰图（Rose Chart）**：利用 `px.bar_polar` 展现市场主流题材的占比，帮助管理员快速识别当前影视库的风格分布。
* **中西审美差异对比图（Cultural Comparison）**：系统通过对齐豆瓣 Top 250 与 IMDb 数据，构建背对背的双向柱状图。左侧展示豆瓣绿（豆瓣评分），右侧展示 IMDb 黄（IMDb 评分），量化展示了中西方观众在经典影片审美上的差异与共识。
* **时代变迁热力图（Evolution Heatmap）**：以年代为 X 轴、题材为 Y 轴，通过颜色的深浅动态展现过去数十年间不同电影类型（如科幻、动作、纪录片）的兴衰演变。

**3. 前端集成与热更新机制**
在 `admin_analytics.py` 页面中，系统利用 NiceGUI 的 `iframe` 组件嵌入上述静态 HTML 图表文件。为了方便管理员获取最新统计数据，页面顶部设有“更新图表缓存”按钮。点击后，系统会在后台重新运行 `generate_charts.py` 脚本，并利用 JavaScript 技巧（添加时间戳 `?v=timestamp`）强制刷新所有 `iframe` 的内容，实现了无需重启服务器的动态数据分析更新。
