### 1.3 设计内容与主要贡献

针对当前影视推荐系统面临的信息过载、推荐维度单一以及缺乏情感共鸣等痛点，本课题设计并实现了一套基于大数据与人工智能的现代化影视推荐系统（MovieRec Sys）。系统深度整合了分布式计算、机器学习、自然语言处理以及全栈Web开发技术。

#### 1.3.1 主要设计内容

本系统的设计内容涵盖了从数据采集、存储、计算到最终交互呈现的完整生命周期，主要包括以下四大核心模块：

**1. 海量异构数据底座与预处理模块**
系统以 PostgreSQL 为核心数据库，接入了千万级别的 IMDb 官方离线数据集（涵盖电影、剧集、演职人员、评分等多维数据）。针对部分缺失的票房数据与中文本地化数据，系统内置了基于 Python 异步机制（Asyncio + HTTPX）的多线程爬虫模块，动态抓取 Box Office Mojo 全球票房数据与豆瓣 Top 250 榜单，构建了结构化、高质量的影视知识库底座。

**2. 多算法融合的混合推荐引擎模块**
为了兼顾推荐的广度（召回率）与精度，系统设计了多级混合推荐架构：

* **离线推荐引擎：** 采用 Apache Spark 框架下的 PySpark 引擎，利用交替最小二乘法（ALS）处理海量用户评分矩阵，挖掘用户与物品的隐含特征（Latent Features），生成离线候选集，解决数据稀疏性问题。
* **实时在线推荐：** 采用基于物品的协同过滤算法（Item-Based CF），通过 Scikit-learn 计算高维余弦相似度矩阵。当用户在前端进行收藏或评分操作时，系统能以毫秒级响应，实时计算并更新个性化推荐列表。
* **冷启动策略：** 针对新注册或无交互记录的用户，系统提供基于 IMDb 热度（投票数）和分类精选的兜底推荐策略。

**3. 基于 NLP 的自然语言情感交互模块**
打破传统推荐系统仅依赖鼠标点击或标签选择的冷冰冰交互方式，系统引入了“情感树洞”功能。后端集成了 Hugging Face 的 `chinese_xlm_xnli` 预训练语言模型，利用零样本分类（Zero-Shot Classification）技术，对用户输入的口语化自然语言（如“今天加班好累，想看点轻松的”）进行情绪意图分析。系统根据识别出的情绪状态（如疲惫、难过、愤怒等），结合预设的心理学治愈策略，智能映射并推荐相应类型的影片，并配以暖心文案，实现人机情感交互。

**4. 影视全域数据分析与可视化大屏模块**
系统不仅服务于普通用户，还为管理员设计了强大的数据可视化大屏。基于 Pandas 数据处理与 Plotly 动态图表技术，系统能够将海量数据转化为直观的商业与文化洞察。可视化内容包括：商业价值与口碑关联的气泡图（ROI Bubble Chart）、影视题材变迁的时空热力图（Evolution Heatmap）、中西审美差异的双向柱状图（Douban vs IMDb）以及题材偏好玫瑰图等。

#### 1.3.2 主要贡献与创新点

本课题的主要贡献与创新之处体现在以下三个方面：

**1. 构建了长短期兴趣结合的混合推荐架构**
传统单一推荐算法容易陷入“信息茧房”或面临严重的“冷启动”问题。本课题的贡献在于成功地将工业界主流的大数据离线计算（Spark ALS）与基于内存矩阵的实时在线计算（Item-CF）深度结合。这种混合架构不仅保证了推荐系统在面对海量历史数据时的泛化能力与覆盖率，同时赋予了系统敏锐捕捉用户即时兴趣漂移的能力，实现了真正意义上的“千人千面”。

**2. 创新性地引入了基于认知智能的情感交互推荐**
本系统的最大亮点在于跨越了传统“基于行为数据”的推荐范畴，迈入了“基于认知与情绪感知”的新领域。通过将前沿的 NLP 大模型（XLM/Transformer）轻量化集成至推荐流水线中，系统被赋予了“共情能力”。这极大提升了推荐系统的温度与用户粘性，为解决用户在“信息过载”且“目标不明确”时的观影选择难题提供了一种全新的、极具人文关怀的解决方案。

**3. 实现了兼具商业洞察与高性能表现的全栈系统方案**
在工程实践层面，本系统摒弃了臃肿的传统 Web 框架，采用了基于 Python 3.11 和 FastAPI + NiceGUI 的现代化异步（Async/Await）全栈架构，配合 PostgreSQL 的异步驱动（asyncpg），大幅提升了系统的并发处理能力与 I/O 吞吐量。此外，系统设计的“数据可视化大屏”不仅是一个功能展示，它通过对千万级冷数据的清洗与多维聚合，直接输出了具有高度可解释性的行业洞察报告（如神作挖掘、票房与口碑的非线性关系等），为平台的内容引进与运营决策提供了可靠的量化依据。
