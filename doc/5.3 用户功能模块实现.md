### 5.3 用户功能模块实现

用户功能模块是影视推荐系统直接面向 C 端受众的交互视窗。本系统基于 NiceGUI 框架构建了响应式的现代化前端界面，将复杂的后端推荐算法与自然语言大模型无缝封装，为用户提供流畅、直观的观影探索体验。本节将详细阐述用户前端核心功能的工程实现细节。

#### 5.3.1 账号鉴权与个性化画像收集

构建精准的个性化推荐系统，首先需要建立完善的用户账号体系与画像基座。本系统的注册与登录模块不仅承担了基础的安全鉴权任务，还巧妙融入了冷启动时期所需的基础画像数据收集。

**1. 安全鉴权机制**
在 `services/auth_service.py` 中，系统实现了高标准的密码安全策略。当用户在注册页面（`/register`）提交表单时，后端并不会直接明文存储密码，而是调用 `passlib` 库的 `bcrypt` 算法生成包含随机盐值（Salt）的哈希密文。在用户登录验证成功后，系统利用 NiceGUI 框架原生的浏览器会话管理机制，将用户的 `user_id`、`username` 和 `role` 写入 `app.storage.user` 字典中，建立起安全的会话级（Session）鉴权上下文，彻底避免了传统 JWT 令牌在单体架构下复杂的签发与吊销管理。

**2. 静态画像标签收集**
在注册界面的 UI 设计上，除了必填的账号密码，系统使用 `ui.select` 等表单组件，鼓励用户选择其性别（Gender）与年龄段（Age Group）。这些静态的人口统计学画像标签被持久化至 PostgreSQL 的 `users` 表中，在用户尚未产生足够多行为数据的“冷启动”阶段，系统可据此辅以粗粒度的规则匹配（如针对不同年龄段分发不同年代的经典高分电影），从而平滑度过信息真空期。

#### 5.3.2 分类过滤与前台瀑布流展示

面对千万级别的影视库，如何让用户高效地检索与浏览内容是前端展示层设计的核心。在 `pages/user_home.py` 中，系统构建了带有动态分类过滤功能的瀑布流展示页。

**1. 声明式分类导航与动态过滤**
系统在页面顶部使用 NiceGUI 的 `ui.tabs` 组件构建了清晰的分类导航栏，包含“全部 (All)”、“电影 (Movie)”、“剧集 (TV)”、“动漫 (Anime)”、“综艺 (Variety)”与“纪录片 (Documentary)”等选项。
当用户切换 Tab 时，前端会触发异步回调函数，将当前选择的 `category` 参数传递给后端的推荐服务。在底层 SQL 查询中（如 `_apply_category_filter` 函数），系统会利用 SQLAlchemy 的 `where` 和 `ilike` 操作符，针对 `titleType` 字段与 `genres` 数组字段进行精准拦截。例如，当选择“动漫”时，系统会在 `where` 子句中强制附加 `genres.ilike('%Animation%')`，确保返回的数据流严格契合用户的频道意图。

**2. 高性能卡片瀑布流渲染**
在获取到后端的电影列表对象后，系统利用 NiceGUI 提供的网格布局组件（如 `ui.grid(columns=4)`）动态渲染电影卡片。每个卡片（`ui.card`）内部集成了 `ui.image`（展示海报）、影视主标题、上映年份标签以及高亮的平均分角标。由于采用的是前后端同构架构，这些 Python 声明式的 UI 代码会在底层直接编译为高效的 Vue.js 虚拟 DOM，保证了在一次性加载大量图片与元素时浏览器的丝滑滚动体验。

#### 5.3.3 TMDB 接口协同的影片详情展示

当用户在瀑布流中点击某一部感兴趣的电影卡片时，系统会通过单页路由（SPA）跳转至专属的影片详情页（`/movie/{tconst}`）。该页面旨在提供极具沉浸感与信息密度的观影参考。

**1. 丰富的基础元数据与剧组图谱**
详情页的后端逻辑不仅会查询 `movie_summary` 表，还会通过 `tconst` 外键向下穿透，联表查询 `title_crew`、`title_principals` 以及 `name_basics` 表。前端界面会结构化地呈现导演、编剧、主演阵容（含饰演的具体角色名），以及通过 `ui.chip` 组件渲染的题材标签，为用户提供详尽的上下文信息。

**2. 外部 API 数据协同与懒加载**
由于 IMDb 官方离线数据缺乏高质量的剧照（Backdrop）与内容简介（Overview），详情页在渲染生命周期的早期，会异步调用 `services/tmdb_service.py` 模块。系统携带影片的 IMDb ID 实时向 TMDB API 发起请求，抓取该片的高清背景图与多语种剧情简介。获取后，不仅立即更新前台界面的大尺寸首屏 Banner，还会将其物理回写至本地数据库进行持久化缓存。这种“按需懒加载 + 永久缓存”的协同机制，既保证了详情页视觉表现力的震撼性，又极大节省了外部 API 的请求配额与加载带宽。

#### 5.3.4 交互反馈：评分与收藏管理

推荐系统不仅是信息的输出方，更是用户意图的收集器。系统在前台提供了直观、轻量的交互组件，以零摩擦的方式捕获用户的实时反馈，这些反馈将直接作为在线 Item-CF 算法调整权重的基准。

**1. 无缝的评分与收藏操作**
在电影卡片与详情页中，系统分别设计了两个核心交互入口：

* **红心收藏组件**：使用带状态切换的 `ui.button(icon='favorite')`。当用户点击时，前端发起异步请求至后端的 `interaction_service.py`。后端在验证用户权限后，向 `user_favorites` 表执行插入操作。成功后，前端图标立刻变为红色实心，实现“即点即反馈”的丝滑体验。
* **滑动评分弹窗**：通过集成 `ui.slider(min=1, max=10, step=0.5)` 滑动条组件，允许用户对影片进行细粒度评价。用户拖动滑块并确认后，数据以 Upsert 方式写入 `user_personal_ratings` 表，更新该用户的偏好历史记录。

**2. 个人中心与历史管理**
为了赋予用户对其个人数据的完全控制权，系统开发了独立的个人中心模块（`/user/favorites` 和 `/user/ratings`）。这些页面利用 NiceGUI 的高级表格或列表组件，按时间倒序展示用户的历史足迹。用户不仅可以在此重温自己看过的佳作，还可以执行“取消收藏”或“修改/删除评分”操作。后端的互动服务会实时响应这些删除指令，并同步重置内存中的协同过滤权重，确保推荐结果能够随用户兴趣的转变而迅速自我修正。
