### 3.3 系统角色与权限设计

为了保障影视推荐系统（MovieRec Sys）的数据安全与业务逻辑的隔离，系统引入了基于角色的访问控制模型（Role-Based Access Control, RBAC）。根据实际业务需求与前后台功能的划分，系统设计了两种核心角色：**普通用户（User）** 和 **超级管理员（Admin）**。通过明确的权限边界划分与统一的路由拦截机制，系统实现了前后端业务的安全隔离。

#### 3.3.1 角色定义与数据模型

在底层数据模型设计中，角色的区分直接体现在 `users` 实体表中。

* **角色字段（Role Field）**：`User` 模型中定义了 `role` 字段（`Column(String, default="user")`），用于显式标识用户的身份级别。
* **密码加密存储**：为了保障所有角色的账户安全，系统在 `auth_service.py` 中集成了 `passlib` 库，采用 `bcrypt` 算法对用户明文密码进行不可逆的哈希（Hash）加密存储（`hashed_password` 字段）。

#### 3.3.2 普通用户（User）权限设计

普通用户是本系统的 C 端（客户端）服务对象，其主要诉求是获取个性化的影视内容与交互体验。

* **账号获取**：普通用户通过前台的注册页面（`/register`）进行账号注册。在注册接口的逻辑中，系统强制将其 `role` 属性设定为 `'user'`，防止越权创建高权限账号。
* **权限范围**：
1. **公共资源访问**：未登录的访客与普通用户均可访问前台首页（`/`）以及电影详情页（`/movie/{tconst}`），浏览分类榜单与影视基础信息。
2. **交互与推荐权限**：登录后的普通用户解锁完整的交互权限，包括对影视作品进行“红心收藏”、提交“1-10分”的滑动打分，以及使用“情感树洞”获取 NLP 定制推荐。
3. **个人数据管理**：拥有专属的个人中心，可访问 `/user/favorites` 和 `/user/ratings` 路由，对其个人的历史收藏与评分记录进行分页查询或物理删除。


* **权限限制**：普通用户绝对禁止访问任何以 `/admin` 开头的后台管理路由。

#### 3.3.3 超级管理员（Admin）权限设计

超级管理员是本系统的 B 端（管理端）核心运营人员，负责整个平台的数据维护、用户管理与算法调度。

* **账号获取**：为了系统安全，超级管理员账号无法通过前端公开注册页面获取。系统提供了一个独立的初始化脚本 `init/create_admin.py`，只有拥有服务器运行权限的开发或运维人员，才能通过终端运行该脚本，绕过常规注册流程，在底层数据库中直接写入 `role='admin'` 的高权限账户。
* **权限范围**：管理员除了拥有普通用户的所有前台功能外，还拥有对系统的全局最高控制权，具体包括：
1. **系统大屏与算法控制**：访问 `/admin` 仪表盘，查看系统全局运行指标，并拥有“一键触发”后台协同过滤推荐模型（Item-CF）重新训练的最高指令权限。访问 `/admin/analytics` 查看深度数据可视化大屏。
2. **核心资源库管理**：拥有对影视主库（`/admin/movies`）、剧集库（`/admin/episodes`）、演职人员库（`/admin/people`）以及剧组关系库（`/admin/crew`）的完全 CRUD 操作权限。支持在修改底层数据后执行“重建缓存”操作，以热更新前端展示数据。
3. **用户与评分监管**：访问 `/admin/users` 审核并修改平台用户信息，进行违规账号的删除或密码重置；访问 `/admin/ratings` 监管影视全局评分体系，对异常刷分行为进行物理删除或数据修正。



#### 3.3.4 权限拦截与安全机制实现

为了在代码逻辑层面严格落实上述权限划分，系统在主路由入口（`main.py`）中实现了一套轻量级且高效的**路由守卫（Router Guard）机制**。

* **会话级状态存储**：在用户成功验证账号密码后，`auth_service.py` 会返回完整的 `User` 对象，系统随即将其 `user_id`、`username`、`role` 以及 `authenticated` (布尔值) 写入 NiceGUI 框架提供的 `app.storage.user` 会话字典中。该字典依赖浏览器的 Cookie/Session 机制进行加密维系。
* **全局鉴权拦截器（Interceptor）**：在 `main.py` 中定义了核心拦截函数 `check_admin_access()`。
1. 首先校验 `app.storage.user.get('authenticated')` 状态，未登录者将被拦截并重定向至 `/login` 页面。
2. 其次校验身份等级，若当前会话的 `role` 属性不等于 `'admin'`，则触发拒绝访问逻辑，系统会弹出“权限拒绝”的警告通知，并将其强制踢回前台首页（`/`）。


* **路由级绑定**：系统对所有后台相关的子页面（如 `@ui.page('/admin')`, `@ui.page('/admin/users')` 等）都在函数执行的第一行硬性绑定了 `if not check_admin_access(): return` 校验逻辑。这种集中式与分散式结合的防御机制，确保了无论用户通过何种方式（如直接输入 URL 或非法抓包）尝试绕过前端界面访问后台接口，都会在服务器处理的第一道防线被成功拦截，保障了系统数据的绝对安全。
